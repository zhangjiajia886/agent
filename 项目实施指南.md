# TTS 应用项目实施指南

## 📋 项目概述

本项目是一个基于 Fish Speech API 的 TTS/ASR 语音服务平台，采用前后端分离架构，预留 RAG、知识图谱和 Agent 扩展能力。

### 技术栈
- **后端**: Python FastAPI + SQLAlchemy + MySQL
- **前端**: Vue3 + TypeScript + Element Plus
- **数据库**: MySQL + Redis + Elasticsearch + Milvus
- **第三方服务**: Fish Speech API

## 🚀 快速启动

### 1. 启动数据库服务

由于网络问题，Docker 镜像拉取可能失败。建议：

**方案 A：配置镜像加速后重试**
```bash
# 参考 Docker镜像加速配置.md 配置镜像加速器
# 然后执行
cd /Users/zjj/home/learn26/ttsapp
docker-compose up -d
```

**方案 B：本地安装（推荐用于开发）**
```bash
# macOS 使用 Homebrew 安装
brew install mysql redis elasticsearch

# 启动服务
brew services start mysql
brew services start redis
brew services start elasticsearch
```

### 2. 初始化数据库

```bash
# 连接 MySQL
mysql -u root -p

# 执行初始化脚本
source /Users/zjj/home/learn26/ttsapp/backend/init_db.sql
```

### 3. 启动后端服务

```bash
cd /Users/zjj/home/learn26/ttsapp/backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # macOS/Linux

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入必要配置

# 启动服务
python -m app.main
```

后端 API 文档: http://localhost:8000/docs

### 4. 启动前端服务

```bash
cd /Users/zjj/home/learn26/ttsapp/frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端访问地址: http://localhost:3000

## 📝 开发指南

### 后端开发

#### 添加新的 API 接口
1. 在 `app/api/v1/` 创建路由文件
2. 在 `app/schemas/` 定义请求/响应模式
3. 在 `app/models/` 添加数据模型（如需要）
4. 在 `app/main.py` 注册路由

#### 数据库迁移
```bash
# 生成迁移文件
alembic revision --autogenerate -m "描述"

# 执行迁移
alembic upgrade head
```

### 前端开发

#### 添加新页面
1. 在 `src/views/` 创建页面组件
2. 在 `src/router/` 添加路由配置
3. 在 `src/api/` 添加 API 调用函数
4. 在 `src/types/` 定义 TypeScript 类型

#### API 调用示例
```typescript
import request from '@/api/request'

export const synthesizeSpeech = (data: TTSRequest) => {
  return request.post('/api/v1/tts/synthesize', data)
}
```

## 🐛 常见问题

### 1. Docker 镜像拉取失败
**解决方案**: 参考 `Docker镜像加速配置.md` 配置镜像加速器

### 2. 数据库连接失败
**检查项**:
- MySQL 服务是否启动
- 数据库用户名密码是否正确
- DATABASE_URL 配置是否正确

### 3. Fish Speech API 调用失败
**检查项**:
- API Key 是否正确
- 网络连接是否正常
- 账户积分是否充足

### 4. 前端无法连接后端
**检查项**:
- 后端服务是否启动（http://localhost:8000）
- CORS 配置是否正确
- Vite 代理配置是否正确
