# ttsapp 前端测试方案与测试用例

## 1. 文档说明

本文档用于指导  前端功能测试、联调验收与回归验证，适用于以下角色：

- **产品经理**：确认交付范围与验收标准
- **测试工程师**：按用例执行功能、异常与回归测试
- **前端/后端开发**：定位页面展示、接口联调、任务轮询与媒体资源访问问题

本文档重点覆盖以下模块：

- **TTS 语音合成**
- **声音模型管理与选择**
- **Podcast 播客语音生成**
- **Singing 歌声合成 / 歌声转换**
- **Digital Human 数字人视频生成**
- **任务状态轮询、结果展示、媒体下载**

## 4. 测试策略

### 4.1 测试层次

- **冒烟测试**：验证页面可打开、核心功能可用
- **功能测试**：验证表单、上传、任务处理、结果展示完整闭环
- **异常测试**：验证空输入、错误文件、接口失败场景
- **回归测试**：验证新改动未破坏既有功能

### 4.2 判定原则

若满足以下条件，可判定单项用例通过：

- **前端无报错崩溃**
- **接口请求符合预期**
- **页面提示准确**
- **轮询逻辑能正常结束**
- **媒体资源可播放或下载**

## 6. 详细测试用例

## 6.1 TTS 页面测试

**页面路径**：`/tts`

### 用例 TTS-01：声音模型列表加载

- **测试目标**：验证声音模型可在下拉框中正确展示
- **前置条件**：数据库中已有声音模型数据
- **测试步骤**：
  1. 打开 `/tts`
  2. 点击“声音模型”下拉框
  3. 观察下拉选项数量与显示内容
- **预期结果**：
  - 可看到已有声音模型
  - 若现网已有 82 个模型，应全部可见
  - 下拉项显示模型标题，无空白项、无重复异常
- **检查重点**：
  - `GET /api/v1/voice-models/?skip=0&limit=100`
  - 返回结构包含 `total` 与 `items`

### 用例 TTS-02：基础语音合成

- **测试目标**：验证 TTS 任务创建、轮询与结果展示闭环
- **测试步骤**：
  1. 输入一段中文文本
  2. 选择任意声音模型
  3. 格式选择 `wav`
  4. 点击“开始合成”
- **预期结果**：
  - 成功提交任务
  - 页面进入处理中状态
  - 任务完成后出现播放器
  - 音频可播放、可下载
  - 历史任务列表新增记录

### 用例 TTS-03：默认声音合成

- **测试目标**：验证不选择声音模型时，系统默认音色可正常工作
- **测试步骤**：
  1. 输入文本
  2. 不选择声音模型
  3. 直接提交合成
- **预期结果**：
  - 请求成功发起
  - 后端不报 `voice_model_id` 为空错误
  - 任务可完成并返回音频

### 用例 TTS-04：空文本校验

- **测试目标**：验证前端阻止无效提交
- **测试步骤**：
  1. 清空输入文本
  2. 点击“开始合成”
- **预期结果**：
  - 按钮不可用或不发起请求
  - 页面无异常报错

### 用例 TTS-05：情感标记测试

- **测试目标**：验证调试区域情感音频播放能力
- **测试步骤**：
  1. 输入带情感标记文本
  2. 点击“播放测试”
- **预期结果**：
  - 返回音频流
  - 页面生成可播放音频
  - 失败时显示明确错误提示

## 10. 已知实现说明

### 10.1 声音模型加载上限

当前 TTS 页面已按 `getModels(0, 100)` 拉取声音模型。

因此：

- **82 个声音模型可完整展示**
- **若未来模型数量超过 100，建议升级为分页或搜索式加载**

### 10.2 SoulTask 字段为空的情况说明

`Podcast`、`Singing`、`Digital Human` 共用 `SoulTask` 通用任务表，不同任务类型仅填充相关字段。

因此：

- **部分字段为空属于正常设计现象**
- **不应直接判定为缺陷**

示例：

- **Podcast** 更关注 `input_text / ref_audio_url / ref_audio2_url`
- **Singing** 更关注 `ref_audio_url / source_audio_url / output_url`
- **Digital Human** 更关注 `ref_image_url / ref_audio_url / output_url`
