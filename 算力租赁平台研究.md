# 算力租赁平台研究报告

> **目标**：临时部署 SoulX-FlashHead（数字人）、SoulX-Singer（歌声合成）、SoulX-Podcast（播客语音）三个模型做实验  
> **核心诉求**：按需开关、保留镜像、价格可控、部署门槛低  
> **首选 GPU**：RTX 5090（32GB GDDR7）  
> **调研时间**：2026-04

---

## 一、三个模型的算力需求分析

在选平台之前，先明确这三个模型分别需要什么级别的 GPU。

| 模型 | 类型 | 最低可跑显存 | 推荐显存 | 主要依赖 |
|------|------|-------------|---------|---------|
| **SoulX-FlashHead** | 视频生成（说话头） | 24GB | **32GB（Lite 最优）** / 48GB+（Pro） | PyTorch, FlashAttention, ffmpeg, wav2vec2 |
| **SoulX-Singer** | 歌声合成 | 16~24GB | **32GB（宽裕）** | PyTorch, 声码器, 音频处理 |
| **SoulX-Podcast** | 播客/TTS 合成 | 12~16GB | **32GB（宽裕）** | PyTorch, FlashCosyVoice, vLLM（可选） |

### 结论

- **三个模型都要跑**：推荐 **1 x RTX 5090（32GB）**，比 24GB 卡多 8GB 余量，FlashHead Lite 更从容
- **RTX 5090 vs RTX 4090**：同为 Ada/Blackwell 架构消费卡，5090 多 8GB 显存 + 算力更强，千样本推理成本比 4090 低约 21%
- **FlashHead Pro 模式**：需要双卡 48GB+，实验阶段建议只用 Lite 模式

### 为什么选 RTX 5090

| 对比项 | RTX 4090 (24GB) | RTX 5090 (32GB) |
|--------|----------------|----------------|
| 显存 | 24GB GDDR6X | **32GB GDDR7** |
| 显存带宽 | 1 TB/s | **1.79 TB/s** |
| CUDA 核心 | 16384 | **21760** |
| FlashHead Lite | 刚好够用 | **有 8GB 余量，更稳** |
| Singer + Podcast | 够用 | **更宽裕，可预加载** |
| 价格（租赁） | ~2 元/hr | ~3 元/hr |
| 性价比 | 高 | **综合更优（算力/显存/时间）** |

### 推荐实验配置

> **1 x RTX 5090（32GB）**  
> 三个模型不同时跑，按需切换，单卡 32GB 完全够用且有余量

### RTX 5090 (32GB) vs A100 (40GB) 对比

价格几乎一样（≈ ¥3/hr），但两张卡的设计哲学完全不同：

| 对比项 | RTX 5090 (32GB) | A100 (40GB) |
|--------|-----------------|-------------|
| **架构** | Blackwell（2025 消费级旗舰） | Ampere（2020 数据中心级） |
| **显存** | 32GB GDDR7 | 40GB HBM2e |
| **显存带宽** | **1.79 TB/s** | 1.56 TB/s |
| **CUDA 核心** | **21760** | 6912 |
| **FP16 算力** | **~100+ TFLOPS** | ~78 TFLOPS（不开稀疏） |
| **FP32 算力** | **~105 TFLOPS** | ~19.5 TFLOPS |
| **功耗** | 575W | 250~300W |
| **租赁价格** | **≈ ¥2.98~3.50/hr** | ≈ ¥3.28/hr |
| **NVLink 多卡互联** | ❌ 不支持 | ✅ 支持 |
| **ECC 内存** | ❌ | ✅ |

#### 针对三个模型的实际影响

| 维度 | RTX 5090 | A100 40GB | 胜者 |
|------|----------|-----------|------|
| **FlashHead Lite（推理）** | 32GB 够用，推理速度快（高频+多核） | 40GB 更宽裕，但推理速度略慢 | **5090**（推理快） |
| **FlashHead Pro** | 32GB 不够（需 48GB+） | 40GB 勉强可试 | **A100**（显存多 8GB） |
| **Singer / Podcast** | 绰绰有余，推理飞快 | 绰绰有余，推理较快 | **5090**（速度优势） |
| **单卡推理性价比** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | **5090** |
| **多卡训练/微调** | 不适合（无 NVLink） | 很适合 | **A100** |
| **同价格下推理吞吐** | 更高 | 较低 | **5090** |

#### 结论

| 场景 | 推荐 |
|------|------|
| **你的场景（单卡推理实验）** | **RTX 5090 ✅** — 同价位推理更快，32GB 够三个模型 |
| 需要尝试 FlashHead Pro | A100 40GB — 多 8GB 显存可以勉强尝试 |
| 多卡训练/微调模型 | A100 — NVLink + ECC 是刚需 |
| 长时间挂机跑批量任务 | A100 — 功耗低（250W vs 575W），服务器更稳定 |

> **一句话**：A100 是「稳重的数据中心老将」，5090 是「暴力的消费级新锐」。  
> 做实验推理选 5090，做训练/长时间生产选 A100。你的场景 → **5090 更合适**。

## 五、关于"保留镜像"的详细说明

这是你最关心的功能之一。

### AutoDL 的镜像机制

```
┌─────────────┐     保存镜像      ┌──────────────┐
│  运行中实例   │ ──────────────→ │  自定义镜像    │
│ (GPU 开机中)  │                 │ (存在云端)     │
└─────────────┘                  └──────────────┘
       │                                │
       │ 关机                           │ 开机时选择
       ▼                                ▼
┌─────────────┐                  ┌──────────────┐
│  已关机实例   │ ←───────────── │  新实例       │
│ (数据保留)   │   用镜像恢复     │ (环境完整)    │
└─────────────┘                  └──────────────┘
```

- **保存镜像**：把已装好的环境（Python 包、模型权重、配置）打包
- **30GB 以下免费**：基础环境通常不超
- **模型权重建议放数据盘**：不占镜像空间
- **下次开机**：选自己的镜像 → 环境秒恢复

### Vast.ai / RunPod 的镜像机制

- 通过 **Docker 镜像** 保存环境
- 需要你自己构建 Dockerfile
- 模型权重通常挂载外部存储
- 门槛比 AutoDL 略高

## 六、部署建议与优先级

### 部署顺序建议

| 优先级 | 模型 | 原因 |
|--------|------|------|
| **第一** | SoulX-Podcast | 最轻量，最容易验证，TTS 类需求最基础 |
| **第二** | SoulX-FlashHead | 数字人是当前最大痛点（HF Space 不稳定） |
| **第三** | SoulX-Singer | 歌声合成可后续补充 |

### 单卡部署策略

由于三个模型不需要同时跑，建议：

```
同一台 RTX 5090 实例（32GB）
  ├── ~/models/podcast/     ← Podcast 模型权重
  ├── ~/models/flashhead/   ← FlashHead 模型权重
  ├── ~/models/singer/      ← Singer 模型权重
  ├── ~/services/
  │   ├── start_podcast.sh
  │   ├── start_flashhead.sh
  │   └── start_singer.sh
  └── ~/logs/
```

- 需要哪个就启动哪个服务
- 不用时停止进程，释放显存
- 三个共享同一张卡，不同时并发
