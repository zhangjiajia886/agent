# AutoDL 数据盘丢失恢复手册

> 适用场景：删除实例/换卡后，使用镜像创建新实例，发现 `/root/autodl-tmp` 为空

---

## 一、为什么会发生这种情况

AutoDL 镜像保存机制：

```
✅ 镜像包含（系统盘）:
  /root/               ← 代码、脚本、配置文件
  /root/miniconda3/    ← conda 基础环境
  ~/services/          ← 服务代码

❌ 镜像不包含（数据盘）:
  /root/autodl-tmp/    ← 每次新实例都是空盘
    ├── envs/          ← conda 虚拟环境（需重装，约 16GB）
    │   ├── env-flashhead/
    │   └── env-podcast/
    └── models/        ← 模型文件（需重下，约 20GB）
        ├── SoulX-FlashHead-1_3B/
        ├── SoulX-Podcast/
        └── wav2vec2-base-960h/
```

**根本原因**：数据盘（`/root/autodl-tmp`）是独立的块存储，不会随镜像保存。  
每次新建实例或换卡，数据盘都是全新的空盘。

---

## 二、如何彻底避免

### 方案 A：不删实例，只关机（推荐）

| 操作 | 数据盘 | 费用 |
|---|---|---|
| 关机（保留实例）| ✅ 保留 | 按存储计费（约 ¥0.03/GB/天）|
| 删除实例 | ❌ 清空 | 无费用 |

**只要不删实例，数据盘永远保留。关机不会丢数据。**

### 方案 B：换卡时做数据迁移

换卡前，用 `rsync` 把老卡数据同步到新卡：

```bash
# 在本地执行（老卡 → 新卡）
rsync -avz --progress autodl-old:/root/autodl-tmp/models/ autodl-new:/root/autodl-tmp/models/
rsync -avz --progress autodl-old:/root/autodl-tmp/envs/ autodl-new:/root/autodl-tmp/envs/
```

### 方案 C：保存镜像前先备份（治标）

在保存镜像之前，把关键路径软链接指向系统盘备份目录，  
但此方案受系统盘容量限制（通常 30~50GB），不推荐。

## 三、万一发生了——一键恢复

### 3.1 前提检查

```bash
ssh autodl-9050   # 连接新实例

# 确认数据盘已挂载
df -h | grep autodl-tmp   # 应有 14TB 左右

# 确认软链接状态
ls -la ~/models            # 应指向 /root/autodl-tmp/models
ls -la ~/services/flashhead/models  # 应指向 ~/models
```

### 3.2 一键恢复脚本（已在本地备份）

```bash
# Step 1: 上传并运行下载脚本（约 20~40 分钟）
scp ~/ttsapp/autoDl部署/download_models_d03.sh autodl-9050:~/
ssh autodl-9050 'bash ~/download_models_d03.sh'

# Step 2: 查看下载进度
ssh autodl-9050 'tmux attach -t download'
# 或查看日志
ssh autodl-9050 'tail -f ~/logs/dl_flashhead.log'

# Step 3: 模型下载完成后，运行环境安装脚本（约 30 分钟）
scp ~/ttsapp/autoDl部署/install_envs_d03.sh autodl-9050:~/
ssh autodl-9050 'tmux new-session -d -s install && tmux send-keys -t install "bash ~/install_envs_d03.sh" C-m'

# Step 4: 查看安装进度
ssh autodl-9050 'tail -f ~/logs/install_envs_d03.log'

# Step 5: 全部完成后启动服务
ssh autodl-9050 'bash ~/start_all.sh'
```

### 3.3 恢复时间参考（RTX 5090 实测）

| 阶段 | 内容 | 耗时 |
|---|---|---|
| Phase 1 | 三模型并行下载（20GB）| 25~40 分钟 |
| Phase 2 | env-flashhead 安装 | 15~20 分钟 |
| Phase 3 | env-podcast 安装 | 10~15 分钟 |
| Phase 4 | 软链接 + 服务启动 | 2 分钟 |
| **合计** | | **约 55~75 分钟** |

## 四、恢复所需的关键信息

### 模型下载源

| 模型 | HuggingFace ID | 目标路径 | 大小 |
|---|---|---|---|
| FlashHead | `Soul-AILab/SoulX-FlashHead-1_3B` | `~/models/SoulX-FlashHead-1_3B` | ~14 GB |
| wav2vec2 | `facebook/wav2vec2-base-960h` | `~/models/wav2vec2-base-960h` | ~1.1 GB |
| Podcast | `Soul-AILab/SoulX-Podcast-1.7B` | `~/models/SoulX-Podcast` | ~5.4 GB |

```bash
# 下载命令（加速镜像）
export HF_ENDPOINT=https://hf-mirror.com
hf download <MODEL_ID> --local-dir <目标路径>
```

### conda 环境规格

| 环境 | 路径 | Python | torch |
|---|---|---|---|
| env-flashhead | `/root/autodl-tmp/envs/env-flashhead` | 3.10 | 2.7.1+cu128 |
| env-podcast | `/root/autodl-tmp/envs/env-podcast` | 3.10 | 2.7.1+cu128 |

### 已知坑（安装时必读）

| # | 问题 | 解决 |
|---|---|---|
| 1 | numpy/sympy 从 pythonhosted.org 下载极慢 | 全程加 `-i https://pypi.tuna.tsinghua.edu.cn/simple` |
| 2 | FlashHead `nvidia-nccl-cu12` 版本冲突 | `grep -v nvidia-nccl-cu12 requirements.txt` 过滤后安装 |
| 3 | `~/models` 是断链（指向 autodl-tmp/models）| 先 `mkdir -p /root/autodl-tmp/models`，不要 `mkdir ~/models` |
| 4 | FlashHead 找不到模型文件 | `ln -sf ~/models ~/services/flashhead/models` |

### 服务公网地址（D03机 / RTX 5090）

| 服务 | 端口 | 公网地址 |
|---|---|---|
| FlashHead | 6006 | `https://u982127-9c41-df2246bb.westd.seetacloud.com:8443` |
| Podcast | 6008 | `https://uu982127-9c41-df2246bb.westd.seetacloud.com:8443` |

### 本地 backend/.env 需同步更新

```bash
# 换卡后必须更新 backend/.env 中的两个地址：
SOUL_FLASHHEAD_SPACE=https://u982127-<新ID>.westd.seetacloud.com:8443
SOUL_PODCAST_URL=https://uu982127-<新ID>.westd.seetacloud.com:8443
# 然后重启后端
cd ~/ttsapp && bash start.sh restart
```
