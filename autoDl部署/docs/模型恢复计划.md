| 类型 | 名称 | 存放路径 | 大小（实测） |

## 二、恢复顺序建议


## 三、Phase 1：下载模型

> ⏱️ 预计时间：20\~40 分钟（视网速）  
> ✅ 可以三个并行下载

```bash
ssh autodl-9050

source ~/miniconda3/etc/profile.d/conda.sh
conda activate base

mkdir -p ~/models
export HF_ENDPOINT=https://hf-mirror.com

# 并行下载（各开一个 tmux 窗口，或后台 &）
tmux new-session -d -s download -x 200 -y 50
tmux rename-window -t download:0 'flashhead'
tmux send-keys -t download:0 "export HF_ENDPOINT=https://hf-mirror.com && source ~/miniconda3/etc/profile.d/conda.sh && conda activate base && hf download Soul-AILab/SoulX-FlashHead-1_3B --local-dir ~/models/SoulX-FlashHead-1_3B 2>&1 | tee ~/logs/dl_flashhead.log" C-m

tmux new-window -t download -n 'wav2vec'
tmux send-keys -t download:1 "export HF_ENDPOINT=https://hf-mirror.com && source ~/miniconda3/etc/profile.d/conda.sh && conda activate base && hf download facebook/wav2vec2-base-960h --local-dir ~/models/wav2vec2-base-960h 2>&1 | tee ~/logs/dl_wav2vec.log" C-m

tmux new-window -t download -n 'podcast'
tmux send-keys -t download:2 "export HF_ENDPOINT=https://hf-mirror.com && source ~/miniconda3/etc/profile.d/conda.sh && conda activate base && hf download Soul-AILab/SoulX-Podcast-1.7B --local-dir ~/models/SoulX-Podcast 2>&1 | tee ~/logs/dl_podcast.log" C-m
```

### 查看下载进度

```bash
# 查看各模型已下载大小
ssh autodl-9050 'du -sh ~/models/*/ 2>/dev/null'

# 查看日志
ssh autodl-9050 'tail -5 ~/logs/dl_flashhead.log ~/logs/dl_wav2vec.log ~/logs/dl_podcast.log'
```

## 四、Phase 2：安装 env-flashhead

> ⏱️ 预计时间：15\~25 分钟  
> ⚠️ 已知坑：需过滤 `nvidia-nccl-cu12`（版本冲突）；用清华镜像

```bash
ssh autodl-9050 'mkdir -p ~/logs && nohup bash ~/install_fh_requirements.sh > ~/logs/install_fh.log 2>&1 &'

# 监控（每2秒刷新）
ssh autodl-9050 'tail -f ~/logs/install_fh.log'
```

`install_fh_requirements.sh` 关键逻辑：
- 创建 `/root/autodl-tmp/envs/env-flashhead`（Python 3.10）
- 安装 `torch==2.7.1+cu128`（清华镜像）
- 过滤 `nvidia-nccl-cu12` 后安装 FlashHead `requirements.txt`

---

## 五、Phase 3：安装 env-podcast

> ⏱️ 预计时间：15\~20 分钟  
> 等 Phase 2 完成后再执行（避免 GPU 内存冲突）

```bash
ssh autodl-9050 'nohup bash ~/install_podcast.sh > ~/logs/install_podcast.log 2>&1 &'

# 监控
```

## 六、Phase 4：创建软链接 + 验证

```bash
# FlashHead 用相对路径 models/，需软链接
ssh autodl-9050 'ln -sf ~/models ~/services/flashhead/models && echo "软链接已建"'

# 验证 FlashHead 环境
ssh autodl-9050 'source ~/miniconda3/etc/profile.d/conda.sh && conda activate /root/autodl-tmp/envs/env-flashhead && python -c "import torch; print(\"FH torch:\", torch.__version__, \"CUDA:\", torch.cuda.is_available())"'

# 验证 Podcast 环境
ssh autodl-9050 'source ~/miniconda3/etc/profile.d/conda.sh && conda activate /root/autodl-tmp/envs/env-podcast && python -c "import torch; print(\"PC torch:\", torch.__version__, \"CUDA:\", torch.cuda.is_available())"'

# 验证模型文件
ssh autodl-9050 'du -sh ~/models/SoulX-FlashHead-1_3B ~/models/wav2vec2-base-960h ~/models/SoulX-Podcast'
```

## 八、已知坑（来自旧卡部署记录）

| # | 问题 | 解决方案 |
|---|---|---|
| 1 | `files.pythonhosted.org` 下载 numpy 超时 | 全部改用 `-i https://pypi.tuna.tsinghua.edu.cn/simple` |
| 2 | FlashHead `nvidia-nccl-cu12` 版本冲突 | `grep -v nvidia-nccl-cu12 requirements.txt` 过滤后安装 |
| 3 | FlashHead 找不到模型（相对路径） | `ln -sf ~/models ~/services/flashhead/models` |
| 4 | github.com 克隆超时 | 改用 `kkgithub.com` 镜像（服务代码已在，无需重克隆）|
| 5 | torch 2.7.1 CUDA 依赖包 >3GB，pip 长时间无输出 | 正常现象，耐心等待；用 `tee` 写日志可观察 |
