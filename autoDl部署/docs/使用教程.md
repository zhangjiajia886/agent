## 二、FlashHead 使用方法

### 1. 打开页面

浏览器访问：`https://u982127-981f-b7a2975a.westd.seetacloud.com:8443`

### 2. 配置模型路径（首次使用）

在页面中找到以下两个路径输入框，确认填写：

| 字段 | 值 |
|---|---|
| `ckpt_dir` | `models/SoulX-FlashHead-1_3B` |
| `wav2vec_dir` | `models/wav2vec2-base-960h` |
| `model_type` | `lite`（单卡）或 `pro`（多卡） |

### 3. 上传素材并生成

1. 上传**人脸图片**（正面，光线均匀）
2. 上传**驱动音频**（WAV / MP3，需要嘴型同步的语音）
3. 点击 **Generate** 按钮
4. 等待生成（首次加载模型约 30 秒，后续推理约 10-60 秒）
5. 下载生成的 `.mp4` 视频

> ⚠️ 生成的视频保存在服务器 `~/services/flashhead/gradio_results/` 目录下

## 三、Podcast API 使用方法

### API 说明

Podcast 是一个 **REST API 服务**，不提供网页界面，通过 HTTP 请求调用。

基础地址：`https://uu982127-981f-b7a2975a.westd.seetacloud.com:8443`

### 查看接口文档

```
https://uu982127-981f-b7a2975a.westd.seetacloud.com:8443/docs
```

> 注：/docs Swagger 页面在 AutoDL 环境下可能因 CDN 问题白屏，改用以下方式测试

### 完整接口列表

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/` | 服务状态 |
| GET | `/health` | 健康检查（含 GPU、模型状态） |
| POST | `/generate` | **同步生成**，等待返回音频 |
| POST | `/generate-async` | **异步生成**，返回 task_id |
| GET | `/task/{task_id}` | 查询异步任务进度 |
| GET | `/download/{filename}` | 下载生成的 wav 音频文件 |

### `/generate` 请求参数

```json
{
  "prompt_texts": ["参考文本1"],         // 与上传音频对应的文字（最多4条）
  "dialogue_text": "要生成的文本内容",   // 单人直接写文本，多人用[S1][S2]标注
  "seed": 1988,                          // 随机种子（可选）
  "temperature": 0.6,                    // 采样温度 0.1-2.0（可选）
  "top_k": 100,                          // Top-K 采样 1-500（可选）
  "top_p": 0.9,                          // Top-P 采样 0.0-1.0（可选）
  "repetition_penalty": 1.25             // 重复惩罚 1.0-2.0（可选）
}
```

### 调用示例（curl 单人声音克隆）

```bash
BASE=https://uu982127-981f-b7a2975a.westd.seetacloud.com:8443

# 同步生成：上传参考音频 + 文本，返回音频文件
curl -X POST "$BASE/generate" \
  -F 'audio_files=@/path/to/reference.wav' \
  -F 'data={"prompt_texts":["参考音频对应文字"],"dialogue_text":"你好，这是克隆的声音"}' \
  -o output.wav

# 异步生成（适合长文本）
curl -X POST "$BASE/generate-async" \
  -F 'audio_files=@/path/to/reference.wav' \
  -F 'data={"prompt_texts":["参考文字"],"dialogue_text":"长文本内容"}' 
# 返回 {"task_id": "xxx"}

# 查询任务状态
curl "$BASE/task/xxx"
# 完成后 result_url 里有下载路径

# 下载结果
curl "$BASE/download/xxx.wav" -o result.wav
```

### 多说话人示例

```bash
curl -X POST "$BASE/generate" \
  -F 'audio_files=@speaker1.wav' \
  -F 'audio_files=@speaker2.wav' \
  -F 'data={"prompt_texts":["说话人1参考文字","说话人2参考文字"],"dialogue_text":"[S1]你好[S2]你也好"}' \
  -o podcast.wav
```

### 健康检查

```bash
curl https://uu982127-981f-b7a2975a.westd.seetacloud.com:8443/health
```

## 四、Singer 使用方法（SSH 隧道）

Singer 运行在端口 7862，需要通过 SSH 隧道访问：

```bash
# 在本地终端执行（保持后台运行）
ssh -L 7862:localhost:7862 autodl -N &

# 然后浏览器访问
open http://localhost:7862
```

> ⚠️ Singer 使用 torch 2.2.0，RTX 5090 CUDA 架构（sm_120）可能不完全兼容，如报错属正常现象

## 五、重启服务（实例重启后）

AutoDL 实例重启后，所有服务需要手动重启。执行以下命令：

```bash
ssh autodl 'bash ~/start_all.sh'
```

**同时还需要**重建 FlashHead 的模型路径软链接（系统盘重置后）：

```bash
ssh autodl 'ln -sf ~/models ~/services/flashhead/models'
```

### 查看服务日志

# Podcast

## 七、常见问题

### Q: FlashHead 报 "No such file or directory: models/xxx"
```bash
ssh autodl 'ln -sf ~/models ~/services/flashhead/models'
ssh autodl 'nohup bash ~/start_all.sh > /dev/null 2>&1 &'
```

### Q: Gradio 页面打不开
检查服务是否在运行：
```bash
ssh autodl 'ps aux | grep gradio_app | grep -v grep'
```
如果没有进程，重启：
```bash
ssh autodl 'bash ~/start_all.sh'
```

### Q: Podcast /docs 白屏
正常现象（Swagger CDN 无法访问），直接调用 API 接口即可。

### Q: 磁盘告警
```bash
# 清理生成的临时视频
ssh autodl 'rm -rf ~/services/flashhead/gradio_results/*'
# 磁盘查看
ssh autodl 'df -h /root/autodl-tmp'
```

### Q: 实例关机后如何恢复
1. 在 AutoDL 控制台启动实例
2. 执行 `ssh autodl 'ln -sf ~/models ~/services/flashhead/models && bash ~/start_all.sh'`
3. 等待约 1 分钟服务启动完成
