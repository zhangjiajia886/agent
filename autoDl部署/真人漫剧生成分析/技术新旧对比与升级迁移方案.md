# 技术新旧对比与升级迁移方案

> 基准时间：2026年4月  
> 评估对象：服务器当前部署的全部技术组件  
> 最终目标：升级后集成到 ttsapp

---

## 一、评估标准

| 等级 | 含义 | 处理建议 |
|---|---|---|
| 🟢 **现代** | 2025年底~2026年的技术，仍是主流或前沿 | 保留，直接集成 |
| 🟡 **可用** | 2024年技术，功能完整但有更优替代 | 可用，有余力时升级 |
| 🔴 **过时** | 2022~2023年技术，已被明显超越 | 重点升级对象 |
| ⚫ **淘汰** | 完全被替代，继续用会拖累体验 | 立即替换 |

## 三、视频生成技术评估

| 技术 | 状态 | 原因 | 升级建议 |
|---|---|---|---|
| **Wan 2.2（当前最新）** | 🟢 现代 | 阶跃星辰2025年末发布，14B参数，动漫视频领先 | 重点保留 |
| **Wan 2.1** | 🟡 可用 | 被2.2覆盖，但部分工作流仍依赖 | 逐步迁移到2.2 |
| **LTX 2.3 22B** | 🟢 现代 | 2026年最新版，速度快、质量高 | 重点保留 |
| LTX 0.9.x / 1.x 系列 | 🔴 过时 | 已被2.x版本明显超越 | 清理旧版LoRA |
| AnimateDiff 相关（`animatediff_models/`目录） | ⚫ 淘汰 | 2022年技术，Wan/LTX完全超越 | 不使用，清理工作流 |
| SVD（Stable Video Diffusion） | ⚫ 淘汰 | 2023年，质量和控制力远不如Wan | 停用 |
| ToonCrafter | 🔴 过时 | 2024年初，动漫插帧工具，已有更好替代 | 降优先级 |

**视频生成核心结论**：
- Wan 2.2 + LTX 2.3 是当前最优组合，保留
- 其余老视频模型（AnimateDiff/SVD）应停止使用

```

Layer 3：路由层（API Routes）
  ─ api/v1/comic.py
  ─ REST API：前端调用的标准接口
  ─ 任务管理：异步提交、状态查询、结果下载

Layer 4：前端页面层（Vue Components）

### 12.5 集成后的完整产品能力

```
ttsapp 集成 ComfyUI 后，用户可以：

1. 上传自己的照片
   → 生成仙侠/动漫/Q版等风格的漫画角色形象

2. 输入剧情文案（支持中文）
   → AI自动生成多格漫剧（4格/8格）
   → 每格画面自动续写场景

3. 一键生成动态漫剧
   → 静态漫画 → Wan I2V → 动态视频

4. 用 ttsapp 现有语音能力配音
