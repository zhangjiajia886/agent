# 真人漫剧生成服务介绍

> 基于 AutoDL bjb2 实例（RTX 5090）的 ComfyUI 服务全貌介绍  
> 适用场景：快速了解服务能力 / 向他人介绍 / 确定集成价值

---

## 一、服务是什么

这是一套部署在云端 GPU 服务器上的**多模态 AI 创作平台**，以 **ComfyUI** 为核心引擎，支持以下核心能力：

- **真人照片 → 漫画/仙侠/动漫风格图像**（保留人脸特征）
- **文字描述 → AI 视频**（秒级到分钟级，2~10秒视频片段）
- **静态漫画图 → 动态视频**（给漫画帧加上自然动作）
- **自然语言编辑图像**（"把背景改成云雾仙山"）
- **快速批量出图**（2~8步，秒级生成）

**硬件**：NVIDIA RTX 5090，32GB 显存，保证高质量和快速推理并行。

## 五、真人漫剧生成完整流程示意

```
第一步：准备素材
  └── 一张真人正面照片（清晰，分辨率 ≥ 512×512）
  └── 故事文案（例：少女进入仙境，遭遇上古神兽）

第二步：生成漫剧帧
  ├── 格1（远景）
  │     输入：人脸照片 + "远处仙山，少女背影，古风场景"
  │     工作流：InstantID + 仙侠LoRA
  │     输出：漫画风格远景图（3~5秒）
  │
  ├── 格2（中景）
  │     输入：同一人脸 + "少女回眸，表情惊讶，神兽显现"
  │     工作流：InstantID + ControlNet姿态
  │     输出：人物动作漫画图（3~5秒）
  │
  ├── 格3（近景对话）
  │     输入：同一人脸 + "近景面孔，坚定表情，法力觉醒"
  │     工作流：InstantID + 特效LoRA
  │     输出：特写表情图（3~5秒）
  │
  └── 格4（特写结局）
        输入：同一人脸 + "飞升姿态，背景绽放光芒"
        工作流：InstantID + 仙侠LoRA
        输出：结局图（3~5秒）

第三步（可选）：视频化
  └── 将每格送入 Wan I2V 工作流
  └── 每格生成 2秒动态视频（3~5分钟/格）
  └── 4格合并 = 约8秒完整动态漫剧

总耗时估计：
  静态4格：约 1~2分钟
  动态4格：约 20~30分钟（Wan I2V × 4次）
```

### 6.2 API 调用（程序集成）

```python
# ComfyUI 提供完整 REST API，无需额外框架
import httpx, json, uuid

BASE_URL = "https://u982127-7772b8fbe6d9.bjb2.seetacloud.com:8443"

# 1. 上传参考图片
async def upload_image(image_bytes: bytes) -> str:
    resp = await httpx.post(f"{BASE_URL}/upload/image",
                            files={"image": ("ref.jpg", image_bytes)})
    return resp.json()["name"]

# 2. 提交工作流
async def submit_workflow(workflow: dict) -> str:
    client_id = str(uuid.uuid4())
    resp = await httpx.post(f"{BASE_URL}/prompt",
                            json={"prompt": workflow, "client_id": client_id})
    return resp.json()["prompt_id"]

# 3. 获取结果
async def get_result(prompt_id: str) -> bytes:
    history = await httpx.get(f"{BASE_URL}/history/{prompt_id}")
    filename = history.json()[prompt_id]["outputs"][...]["images"][0]["filename"]
    image = await httpx.get(f"{BASE_URL}/view", params={"filename": filename})
    return image.content
```

## 七、集成到 ttsapp 的价值

### 7.1 为 ttsapp 新增的能力

```
现有 ttsapp 功能：
  TTS语音合成 + ASR语音识别 + AI陪聊 + 播客合成 + 歌声生成 + 数字人视频

集成 ComfyUI 后新增：
  ✅ 真人漫剧生成
     用户上传照片 → 选择风格（仙侠/Q版/水墨）→ 输入剧情文案
     → 自动生成多格漫剧图像
     → 可选：转为动态视频 + TTS配音（打通现有语音能力）
  
  ✅ 漫剧视频生成
     漫画帧 + Wan I2V = 动态漫剧
     + TTS配音（现有） + 背景音乐 = 完整漫剧短片
  
  ✅ AI形象定制
     用户照片 → 专属漫画角色 → 用于数字人工作流（FlashHead）
```

### 7.2 集成优先级建议

| 优先级 | 功能 | 理由 |
|---|---|---|
| P1 | 真人→漫画静图（InstantID + 仙侠LoRA） | 最直接的差异化功能，用户感知最强 |
| P2 | Qwen 自然语言图像编辑 | 门槛低，用户体验直观 |
| P3 | 漫剧静图+TTS配音组合 | 打通现有TTS能力，形成完整产品 |
| P4 | Wan I2V 动态化 | 计算成本较高，作为高级功能 |

## 八、服务局限与注意事项

| 局限 | 说明 |
|---|---|
| **单用户瓶颈** | ComfyUI 默认单队列，并发请求会排队，生产环境需考虑多实例或任务队列 |
| **视频耗时长** | Wan I2V 每段 2 秒视频需 3~6 分钟，不适合实时响应场景 |
| **显存上限** | RTX5090 32GB，同时跑多个大模型会 OOM，需按工作流切换模型 |
| **人脸保持非完美** | InstantID 对侧脸、戴眼镜、遮挡等情况效果下降 |
| **语言偏差** | 部分 Checkpoint 对中文提示词理解差，建议统一用英文提示词 |
