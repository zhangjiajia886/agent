# 数据库和后端实现方案

> 基于《自动化漫剧 Agent 设计》十三章数据库 ER 图 + 现有 ttsapp 后端架构
> 目标：第一版可运行后端，支持前端 Agent 对话页面的全部交互

---

## 一、实施范围

### V1 做什么

| 模块 | 内容 |
|---|---|
| 数据库模型 | model_config、tool_registry、workflow_template、agent_conversation、agent_message（5 张表） |
| REST API | 模型/工具/工作流 CRUD + 会话管理 |
| WebSocket | `/api/v1/comic-agent/ws/chat` 流式 Agent 对话 |
| Agent 核心 | Mock 模式 — 模拟分析、tool_start/tool_done 事件流 |
| 种子数据 | 6 个模型 + 8 个工具 + 8 个工作流模板的初始数据 |

### V1 不做

- generation_task / artifact / comic_project / comic_frame（V2 对接真实 ComfyUI 时加入）
- tool_model_binding / tool_workflow_binding 关联表（V2）
- 真实 Claude API 调用（V2）

## 三、数据库表设计（V1 精简版）

在现有 MySQL 数据库中添加 5 张表，通过 SQLAlchemy `Base.metadata.create_all` 自动建表。

### 3.1 model_config

管理所有模型配置（替代 .env 硬编码）。

### 3.2 tool_registry

Agent 可调用的工具注册表。

### 3.3 workflow_template

ComfyUI 工作流模板（不含 workflow_json，V1 存标识名即可）。

### 3.4 agent_conversation

Agent 对话会话（独立于现有 chat_sessions，前缀 agent_ 避免冲突）。

### 3.5 agent_message

对话消息，支持 role = user / assistant / tool_call / tool_result / system。

## 四、API 设计

### 4.1 配置管理（REST）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v1/comic-agent/models` | 模型列表 |
| PUT | `/api/v1/comic-agent/models/{id}` | 更新模型（启停等） |
| GET | `/api/v1/comic-agent/tools` | 工具列表 |
| PUT | `/api/v1/comic-agent/tools/{id}` | 更新工具（启停等） |
| GET | `/api/v1/comic-agent/workflows` | 工作流列表 |
| PUT | `/api/v1/comic-agent/workflows/{id}` | 更新工作流 |

### 4.2 会话管理（REST）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v1/comic-agent/conversations` | 会话列表 |
| POST | `/api/v1/comic-agent/conversations` | 创建会话 |
| GET | `/api/v1/comic-agent/conversations/{id}` | 获取会话+消息 |
| DELETE | `/api/v1/comic-agent/conversations/{id}` | 删除会话 |

### 4.3 Agent 对话（WebSocket）

```text
WS /api/v1/comic-agent/ws/chat?conversation_id=xxx&token=yyy

客户端 → 服务端:
{ "message": "...", "style": "auto", "frames": 4, "model": "claude-sonnet-4-6", "tts": false, "auto_video": false }

服务端 → 客户端 (JSON 逐条推送):
{ "type": "text", "content": "好的！我来分析..." }
{ "type": "tool_start", "tool": "generate_image", "input": {...} }
{ "type": "tool_done", "tool": "generate_image", "result": "..." }
{ "type": "error", "content": "..." }
{ "type": "done" }
```
