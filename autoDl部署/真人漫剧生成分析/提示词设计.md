## 五、提示词设计规范

### 5.1 通用结构

```
[角色/主体描述], [风格关键词], [动作/表情/状态], [服装/道具], [场景/背景], [光线/氛围], [质量修饰词]
```

### 5.2 风格专项提示词

#### 仙侠漫剧风（配合 `仙侠风格.safetensors`）

### 5.3 人脸保持工作流提示词策略

使用 **InstantID** 或 **IP-Adapter FaceID** 时，提示词需配合人脸图像：

```
# 核心原则：提示词描述"应该是什么样子"，人脸图像负责"保持谁的样子"

# 建议权重设置：
# - IP-Adapter 强度：0.8~0.9（过高会失真）
# - ControlNet 强度：0.5~0.7（保持姿态但不锁死）
# - LoRA 强度：0.6~0.8

正向 Prompt（最小化人脸描述词，让 IP-Adapter 决定）：
"xianxia style, ancient chinese costume, dramatic lighting, 
cinematic composition, masterpiece, best quality"

避免在 Prompt 中指定具体人脸特征（发色、肤色等），
否则会与 IP-Adapter 产生冲突。
```

### 5.4 Qwen 图像编辑提示词（`QwenImage图像系列` 工作流）

"将人物眼睛改为蓝色"
"增加飘逸的长发效果"

```

### 5.5 视频生成提示词（Wan 系列）

视频提示词需要强调**运动描述**：

```

hair flowing in wind, light magic particles swirling,

## 六、关键工作流使用指南

### 6.1 双截棍工作流（FLUX + nunchaku）

```
适用场景：高质量单图生成，RTX5090 可在 ~10s 内完成
路径：~/ComfyUI/user/default/workflows/双截棍-RTX5090/

核心节点：
- NunchakuFluxDiTLoader：加载量化 FLUX 模型
- NunchakuTextEncoderLoader：加载 T5/CLIP 编码器
- KSampler 或 FluxSampler

推荐参数：
- Steps: 20~28（双截棍加速后速度快，可多跑步数）
- CFG: 3.5~7.0（FLUX 风格偏低CFG）
- Sampler: euler / dpmpp_2m
- Scheduler: karras / simple
```

### 6.2 Z-Image Turbo 工作流

```
适用场景：快速预览，2~8步即可出图
路径：~/ComfyUI/user/default/workflows/Z-Image系列/

推荐参数：
- Steps: 4~8
- CFG: 1.0~2.0（极低CFG）
- LoRA: Z-Image-Fun-Lora-Distill-4-Steps-2603-ComfyUI.safetensors
```

### 6.3 Wan 2.2 动漫视频工作流

```
工作流：WAN 2.2-动漫-4090-工作流.json
适用场景：生成动漫风格视频（T2V/I2V）

推荐参数：
- Steps: 20~30
- 分辨率: 480x832 或 832x480（横/竖屏）
- 帧数: 49 或 81（约2~4秒）
- LoRA: Wan2.2_Remix + WanAnimate_relight
```

## 七、漫剧生成实战策略

### 7.1 "真人→漫剧" 转换流程

```
Step 1：准备真人参考图（正面清晰，1:1或4:3比例）
Step 2：使用 InstantID 工作流，保留人脸特征
Step 3：通过 LoRA（仙侠风格/国漫等）控制目标风格
Step 4：用 OpenPose ControlNet 控制姿态（可自定义动作）
Step 5：生成静态漫画帧
Step 6（可选）：送入 Wan I2V 工作流，生成动态版本
```

### 7.2 分格漫画策略

```
方案A：单独生成每帧，保证风格一致性
  - 固定：相同的 Checkpoint + LoRA 组合
  - 固定：相同的 Seed（保持背景一致）或 相同的角色 IP-Adapter 权重
  - 变化：姿态 ControlNet（每帧不同动作）
  - 变化：Prompt 动作描述部分

方案B：使用 Z-Image 批量生成（快速出图）
  - 先用 4步 快速验证构图
  - 满意后提升到 20步 精细化出图
```

## 九、常见问题 & 注意事项

| 问题 | 原因 | 解决方案 |
|---|---|---|
| VRAM OOM（显存不足） | 模型过大 | 切换到 fp8 版本；使用双截棍节点量化 |
| 人脸变形 | IP-Adapter 强度过高 | 降低到 0.7~0.8；检查 insightface 模型是否加载 |
| 风格不突出 | LoRA 强度不够 | 提升到 0.8~1.0；检查 LoRA 与 Checkpoint 兼容性 |
| 视频抖动 | 帧间一致性不足 | 降低 Steps；使用 CausVid LoRA 加速推理 |
| Qwen 编辑效果差 | 指令不精确 | 改用中文指令；更细化描述编辑部位 |
