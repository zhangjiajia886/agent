### 2.2 ComfyUI 是什么

ComfyUI 是一个**基于节点图的 AI 图像/视频生成框架**。

```
工作原理：
用户设计一个 JSON 工作流（节点图）
  → 每个节点 = 一个操作（加载模型 / 采样 / ControlNet / LoRA ...）
  → 节点之间连线 = 数据流动
  → 整个图执行一遍 = 完成一次生成任务

与普通 Stable Diffusion WebUI 的区别：
  WebUI：固定界面，参数有限，适合入门
  ComfyUI：完全自由，节点可任意组合，适合复杂工作流
```

**核心概念**：

| 概念 | 说明 |
|---|---|
| **Checkpoint** | 基础大模型（决定风格基础，如写实/动漫） |
| **LoRA** | 轻量风格/角色微调，叠加在 Checkpoint 上 |
| **ControlNet** | 结构控制（姿态/线稿/深度），让生成内容遵循结构 |
| **IP-Adapter** | 图像提示适配器，让生成内容继承参考图的风格/人脸 |
| **Sampler/Scheduler** | 采样器配置，决定生成质量和速度的平衡 |
| **工作流 JSON** | 节点图序列化文件，可直接加载执行 |

### 2.3 这台机器的技术栈

```

```

## 四、核心功能深度分析

### 4.1 功能A：双截棍 FLUX 图像生成

**原理**：
```
FLUX.1 是 Black Forest Labs 发布的新一代扩散模型（2024年）
  → 比 SD3 更强的文本理解和图像质量
  → 参数量庞大（Dev版约12B）
  
双截棍（nunchaku）= 对 FLUX 的 INT4 量化推理优化
  → 显存从 ~24GB 降至 ~12GB
  → RTX5090 上可达实时推理速度
  → 质量损失极小（量化误差补偿技术）

流程：
文本提示词
  → T5/CLIP 文本编码器
  → FLUX DiT（扩散 Transformer，量化版）
  → VAE 解码
  → 输出图像
```

**工作流路径**：`~/ComfyUI/user/default/workflows/双截棍-RTX5090/`

### 4.2 功能B：Qwen 图像编辑

**原理**：
```
Qwen-Image-Edit 是阿里开源的多模态图像编辑模型（2025年）
  → 输入：原图 + 自然语言指令
  → 输出：编辑后的图像
  
与传统 Inpainting 的区别：
  传统：需要用户画 Mask，指定编辑区域
  Qwen：直接用文字描述，模型自动理解编辑区域和意图

技术链路：
原图 + "将背景改为仙山" 
  → Qwen 图像理解（Florence2/CLIP 辅助）
  → 扩散模型局部重绘
  → 输出编辑结果

可用的 Qwen LoRA（不同编辑能力）：
  - Anything2RealAlpha：将任意图像转为真实感
  - Fusion：多图融合
  - Light-Migration：光线迁移
  - Relight：重新打光
  - Multiple-angles：多角度视图生成
```

### 4.3 功能C：人脸保持生成（真人漫剧核心）

**原理**：
```
InstantID（最强方案）：
  → 输入：参考人脸图 + 文本提示词
  → 人脸特征提取（InsightFace → 512维 Embedding）
  → ControlNet（面部关键点对齐）
  → IP-Adapter（风格/纹理注入）
  → 输出：保留人脸特征的生成图

技术组合：
  InsightFace（人脸检测+特征提取）
  + control_instant_id_sdxl.safetensors（面部关键点）
  + IP-Adapter（人脸 Embedding 注入到 Cross-Attention）
  → 同时保持：面部结构 + 肤色 + 五官比例

IP-Adapter FaceID（轻量方案）：
  → 仅使用 FaceID LoRA（ip-adapter-faceid-plusv2_sd15_lora）
  → 无需 ControlNet，更灵活但人脸保持弱一些
  → 适合风格化较强的场景（人脸特征保持约70%）
```

**这是"真人漫剧"的核心功能**——保留真人面部特征，同时转换为漫画/仙侠风格。

### 4.4 功能D：Wan 视频生成

**原理**：
```
Wan（万象）= 阶跃星辰开源的视频生成模型（2025年）
  → Wan 2.1：14B参数，支持T2V和I2V
  → Wan 2.2：更新版，更好的动漫风格支持
  
T2V（文生视频）：
  文本提示词 → 视频扩散模型 → 若干帧图像序列 → 视频

I2V（图生视频）：
  参考图像 + 运动提示词 → 以参考图为第一帧 → 生成后续帧

技术特点：

    → 送入 Wan I2V 工作流
    → 生成 2~4 秒的动态版本
    → 多段拼接 = 完整漫剧片段
```

## 五、真人漫剧生成完整路径

### 5.1 目标定义

"真人漫剧" = 保留真实人物面部特征，以漫画/仙侠/动漫风格呈现，具有分格叙事结构的图像或视频内容。

### 5.2 生成路径（单格漫画帧）

```
输入素材
  ├── 真人照片（参考脸，1张）
  └── 文字描述（场景/动作/情绪）

Step 1：预处理
  → InsightFace 提取人脸 Embedding
  → OpenPose 提取人体姿态（可选，用于控制动作）

Step 2：风格化生成（InstantID 工作流）
  → 基础模型：NetaYumev35（动漫）或 majicMIX_realistic（写实漫）
  → LoRA：仙侠风格（0.7）或 blindbox_v1_mix（Q版）
  → InstantID：保留人脸特征
  → ControlNet OpenPose：控制姿态（可选）
  → Prompt：[风格] + [场景描述] + [质量词]

Step 3：精修（可选）
  → 高清放大（使用 Upscale 模型）
  → 局部修复（Impact Pack 面部修复）

输出：单格漫画图（768×1024 推荐）
```

### 5.3 生成路径（多格连续漫剧）

```
目标：4格漫剧（起承转合叙事结构）

每格独立生成，通过以下方式保证一致性：
  ├── 固定 Seed（背景/场景一致）
  ├── 固定 IP-Adapter 权重（人脸一致）
  └── 固定 LoRA 组合（风格一致）

格1（起）：远景，交代场景
格2（承）：中景，人物出场动作
格3（转）：近景，情节转折/对话
格4（合）：特写，情绪表达/结局

可选：将静态格送入 Wan I2V → 生成动态漫剧视频
```

## 六、ComfyUI API 接口（集成到 ttsapp 的技术基础）

### 6.1 核心 HTTP API

ComfyUI 提供完整的 REST API，不需要 Gradio Client，直接用 HTTP 调用：

```python
# 基础 URL
COMFYUI_URL = "http://127.0.0.1:6006"  # 服务器本机
# 或从外部访问：
COMFYUI_URL = "https://u982127-7772b8fbe6d9.bjb2.seetacloud.com:8443"

# 核心 API 端点
GET  /object_info          → 查询所有可用节点及参数
GET  /history/{prompt_id}  → 查询任务执行结果
GET  /queue                → 查询当前队列状态
POST /prompt               → 提交工作流执行任务
POST /upload/image         → 上传输入图像
GET  /view?filename=xxx    → 下载输出图像/视频
WS   /ws?clientId=xxx      → WebSocket 实时进度推送
```

### 6.2 调用工作流的标准流程

```python
import httpx
import json
import uuid

COMFYUI_URL = "https://u982127-7772b8fbe6d9.bjb2.seetacloud.com:8443"

async def run_comfyui_workflow(workflow_json: dict) -> dict:
    client_id = str(uuid.uuid4())
    
    # Step 1: 提交工作流
    payload = {
        "prompt": workflow_json,
        "client_id": client_id
    }
    response = await httpx.post(f"{COMFYUI_URL}/prompt", json=payload)
    prompt_id = response.json()["prompt_id"]
    
    # Step 2: 轮询结果（或用 WebSocket）
    while True:
        history = await httpx.get(f"{COMFYUI_URL}/history/{prompt_id}")
        data = history.json()
        if prompt_id in data:
            outputs = data[prompt_id]["outputs"]
            return outputs  # 包含图像/视频文件名
        await asyncio.sleep(1)

async def upload_image(image_bytes: bytes, filename: str) -> str:
    files = {"image": (filename, image_bytes, "image/png")}
    response = await httpx.post(f"{COMFYUI_URL}/upload/image", files=files)
    return response.json()["name"]  # 返回服务器端文件名

async def download_result(filename: str) -> bytes:
    response = await httpx.get(f"{COMFYUI_URL}/view", 
                                params={"filename": filename})
    return response.content
```

### 6.3 工作流 JSON 参数注入模式

ComfyUI 工作流是 JSON 格式的节点图，集成时需要**动态修改节点参数**：

```python
import copy

def inject_params(workflow: dict, overrides: dict) -> dict:
    """
    overrides = {
        "6": {"text": "新的提示词"},          # 节点ID 6 的 text 参数
        "13": {"seed": 42},                    # 节点ID 13 的 seed
        "4": {"ckpt_name": "flux1-dev-fp8.safetensors"}  # 切换模型
    }
    """
    w = copy.deepcopy(workflow)
    for node_id, params in overrides.items():
        for key, value in params.items():
            w[node_id]["inputs"][key] = value
    return w

# 使用示例
workflow = json.load(open("双截棍工作流.json"))
workflow = inject_params(workflow, {
    "正向提示词节点ID": {"text": user_prompt},
    "Seed节点ID": {"seed": random_seed},
})
result = await run_comfyui_workflow(workflow)
```

### 6.4 WebSocket 实时进度

```python
import websockets

async def watch_progress(client_id: str):
    uri = f"ws://127.0.0.1:6006/ws?clientId={client_id}"
    async with websockets.connect(uri) as ws:
        async for message in ws:
            data = json.loads(message)
            if data["type"] == "progress":
                step = data["data"]["value"]
                total = data["data"]["max"]
                print(f"进度: {step}/{total}")
            elif data["type"] == "executed":
                print("节点执行完成:", data["data"]["node"])
            elif data["type"] == "execution_success":
                print("工作流完成！")
                break
```

### 7.2 后端新增模块

参照已有的 `gradio_client.py` 模式，新建 `core/comfyui_client.py`：

```python
# backend/app/core/comfyui_client.py

class ComfyUIClient:
    """ComfyUI HTTP API 封装，复用现有 httpx + BackgroundTask 模式"""
    
    def __init__(self, base_url: str):
        self.base_url = base_url  # settings.COMFYUI_URL
    
    async def generate_image(
        self,
        workflow_name: str,    # 工作流名称（对应预设JSON）
        prompt: str,           # 正向提示词
        negative_prompt: str,

    async def generate_video(
        self,
        workflow_name: str,

        frames: int = 49,
```

### 11.4 测试问题排查指南
