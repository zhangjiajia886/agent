# AutoDL 部署实施计划

> 基于：`AutoDL部署实施设计.md`
> 当前状态：FlashHead 已完成下载，其他模型与服务尚未完成

## 1. 目标

将当前 `ttsapp` 的数字人相关能力从 HuggingFace Spaces 切换到 AutoDL 自建 Gradio 服务，目标模型为：

- `SoulX-FlashHead-1_3B`
- `facebook/wav2vec2-base-960h`
- `Soul-AILab/SoulX-Podcast-1.7B`
- `Soul-AILab/SoulX-Singer`

## 2. 当前实际进度

### 已完成

- **SSH 免密登录** 已配置完成
- **实例基础环境** 已确认
- **GPU / CUDA / PyTorch 基础镜像** 已确认可用
- **工作目录** 已创建：`~/models`、`~/services`、`~/logs`

## 3. 推荐执行顺序

## Phase A：完成模型下载

建议在 `tmux` 会话中执行：

```bash
source ~/miniconda3/etc/profile.d/conda.sh
conda activate base
export HF_ENDPOINT=https://hf-mirror.com
```

下载命令：

```bash
hf download facebook/wav2vec2-base-960h --local-dir ~/models/wav2vec2-base-960h
hf download Soul-AILab/SoulX-Podcast-1.7B --local-dir ~/models/SoulX-Podcast
hf download Soul-AILab/SoulX-Singer --local-dir ~/models/SoulX-Singer
```

## Phase B：创建运行环境

```bash
source ~/miniconda3/etc/profile.d/conda.sh
conda create -n soul python=3.10 -y
conda activate soul
pip install torch==2.7.1 torchvision==0.22.1 --index-url https://download.pytorch.org/whl/cu128
pip install flash_attn==2.8.0.post2 --no-build-isolation
pip install sageattention==2.2.0 --no-build-isolation
pip install gradio>=4.0.0
apt-get update && apt-get install -y ffmpeg tmux git
```

## Phase C：拉取服务代码

```bash
cd /tmp && git clone https://github.com/Soul-AILab/SoulX-FlashHead.git
cd /tmp && git clone https://github.com/Soul-AILab/SoulX-Podcast.git
cd /tmp && git clone https://github.com/Soul-AILab/SoulX-Singer.git
```

安装依赖并复制：

```bash
source ~/miniconda3/etc/profile.d/conda.sh
conda activate soul
cd /tmp/SoulX-FlashHead && pip install -r requirements.txt && cp -r /tmp/SoulX-FlashHead ~/services/flashhead
cd /tmp/SoulX-Podcast && pip install -r requirements.txt && cp -r /tmp/SoulX-Podcast ~/services/podcast
cd /tmp/SoulX-Singer && pip install -r requirements.txt && cp -r /tmp/SoulX-Singer ~/services/singer
```

## Phase D：启动服务

### FlashHead

```bash
conda activate soul
cd ~/services/flashhead
python gradio_app_streaming.py --ckpt_dir ~/models/SoulX-FlashHead-1_3B --wav2vec_dir ~/models/wav2vec2-base-960h --server_port 7860 --server_name 0.0.0.0
```

### Podcast

```bash
conda activate soul
cd ~/services/podcast
python gradio_app.py --model_dir ~/models/SoulX-Podcast --server_port 7861 --server_name 0.0.0.0
```

### Singer

```bash
conda activate soul
cd ~/services/singer
python gradio_app.py --model_dir ~/models/SoulX-Singer --server_port 7862 --server_name 0.0.0.0
```

## Phase E：端口映射与本地联调

在 AutoDL 控制台创建：

- `7860` → FlashHead
- `7861` → Podcast
- `7862` → Singer

然后修改本地 `backend/.env`：

```env
SOUL_FLASHHEAD_SPACE=https://<mapped-7860>
SOUL_SINGER_SPACE=https://<mapped-7862>
SOUL_PODCAST_SPACE=https://<mapped-7861>
SOUL_HF_TOKEN=
```

## Phase F：固化与重启策略

- 生成 `~/start_all.sh`
- 用 `tmux` 管理 3 个服务
- 跑通后保存 AutoDL 平台镜像

## 4. 风险点

- **系统盘空间不足**：当前模型放在 `/root/models`，需要持续关注系统盘占用
- **SSH 中断**：长时间任务必须托管到 `tmux`
- **依赖兼容性**：`flash_attn`、`sageattention`、仓库 requirements 可能出现版本冲突
- **模型真实体积大于预估**：FlashHead 实际体积已明显高于设计文档中的估值

## 5. 重启后简化流程

若部署最终完成并保存镜像，未来理想流程应为：

```bash
ssh autodl
source ~/miniconda3/etc/profile.d/conda.sh
~/start_all.sh
```

然后：

- 查看 `tmux attach -t soul`
- 查看 `nvidia-smi`
- 使用端口映射 URL 做本地联通测试
