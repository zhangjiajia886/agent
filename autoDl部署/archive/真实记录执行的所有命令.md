## 7. 本次遇到的真实问题

### 问题 1：`python` 不在默认 PATH

现象：

```bash
bash: line 1: python: command not found
```

原因：

- AutoDL 实例虽然安装了 miniconda，但未自动激活 base 环境

处理：

```bash
source ~/miniconda3/etc/profile.d/conda.sh
conda activate base
```

### 问题 2：PyTorch 查询显存属性名写错

现象：

```bash
AttributeError: 'torch._C._CudaDeviceProperties' object has no attribute 'total_mem'. Did you mean: 'total_memory'?
```

处理：

- 将 `total_mem` 改为 `total_memory`

### 问题 3：`huggingface-cli` 已弃用

现象：

- 安装 `huggingface_hub` 后提示 `huggingface-cli` 已弃用

处理：

- 改用 `hf download`

### 问题 4：SSH 长连接中断

现象：

```bash
Connection to connect.westd.seetacloud.com closed by remote host.
```

影响：

- 本地命令结束，但远程下载进程仍继续运行

结论：

- 后续长任务必须放入 `tmux` 或 `screen`

### 问题 5：`watch` 不能通过 `ssh autodl "..."` 直接使用

现象：

```bash
Error opening terminal: unknown.
```

原因：

- `watch` 依赖交互式终端

正确做法：

```bash
ssh autodl
watch -n 2 '...'
```

### 问题 6：终端出现 `^[[200~`

原因：

- 粘贴时带入 bracketed paste 控制字符

处理：

- 重输命令即可，不影响下载状态

### 问题 7：出现 `>` 提示符

原因：

- 输入了未闭合引号命令，如：

```bash
hf download" | grep -v grep
```

处理：

```bash
Ctrl + C
```
