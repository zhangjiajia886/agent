# 迁移到 A100 SXM4 40GB 实施方案

> 编写时间：2026-04-25（更新：19:00）  
> 当前实例：RTX 5090 32GB / 150GB数据盘（34%，51GB已用）  
> 目标实例：A100 SXM4 40GB / 150GB数据盘

---

## 重要说明：镜像快照范围

| 内容 | 位置 | 镜像是否保存 |
|---|---|---|
| 服务代码 `~/services/` | 系统盘 overlay | ✅ 保存 |
| 脚本 `~/start_all.sh` 等 | 系统盘 overlay | ✅ 保存 |
| miniconda3 base | 系统盘 overlay | ✅ 保存 |
| **conda envs** (`env-flashhead` 等) | **数据盘 autodl-tmp** | ❌ **不保存** |
| **模型文件** (28GB) | **数据盘 autodl-tmp** | ❌ **不保存** |

**结论：镜像只保存代码，模型和 conda 环境需要单独迁移。**

---

## 迁移方案选择

### 方案A（推荐）：双开实例 rsync 内网迁移

```
旧实例（保持运行）─rsync→ 新实例（A100）
       ↓ 传输完成后
旧实例关机释放
```

- **优点**：数据直接迁移，避免重新下载 28GB 模型
- **缺点**：迁移期间双开，多计费约 1 小时
- **速度**：AutoDL 内网 100MB/s+，51GB 约 8-15 分钟

### 方案B：重新部署（不推荐）

新实例上从零开始，重新下载模型 + 重装环境。
- 耗时：模型下载 1-2 小时 + 环境安装 1 小时 = 2-3 小时
- 适合：旧实例已关机无法访问时

## 方案A 详细步骤

### Step 1：保存镜像快照（5分钟，在旧实例上操作）

```
AutoDL 控制台 → 容器实例 → 当前实例行 → 「更多」→ 「保存镜像」
→ 填写镜像名：soulx-deploy-v1
→ 等待约 5 分钟，状态变为「保存成功」
```

> ⚠️ 镜像会保存：服务代码、安装脚本、start_all.sh、miniconda base

---

### Step 2：开新实例（5分钟）

```
AutoDL 控制台 → 「租用新实例」
→ GPU：A100 SXM4 40GB（搜索 A100）
→ 数据盘：100 GB
→ 镜像：「我的镜像」→ 选 soulx-deploy-v1
→ 地区：选与旧实例相同区域（同区内网传输更快）
→ 确认开机
```

记下新实例的 SSH 登录命令，配置到本地 `~/.ssh/config`：

```
Host autodl-new
  HostName <新实例IP>
  Port <新实例端口>
  User root
  IdentityFile ~/.ssh/id_rsa
```

---

### Step 3：rsync 数据迁移（核心步骤，约 10-20 分钟）

#### 原理说明

```
你的电脑（本地）
    │
    ├─SSH─→ 旧实例（RTX 5090）  ← 数据源：51GB
    │              │
    │     内网直连（100MB/s+）
    │              ↓
    └─SSH─→ 新实例（A100）       ← 数据目标
```

新实例需要 SSH 进旧实例才能 rsync，通过 **SSH Agent 转发** 实现（不需要在服务器间复制密钥）。

#### 第一步：获取旧实例内网 IP（本地执行）

```bash
# 获取旧实例内网 IP（通常是 192.168.x.x 或 10.x.x.x）
ssh autodl 'hostname -I'
# 示例输出：192.168.0.101 172.17.0.1
# 取第一个 IP，记下来，下面要用
```

#### 第二步：把公钥复制到新实例（本地执行）

```bash
# 把本地 SSH 公钥也加到新实例的 authorized_keys
# 这样新实例就能用同一个密钥连旧实例
cat ~/.ssh/id_rsa.pub | ssh autodl-new 'cat >> ~/.ssh/authorized_keys'
```

#### 第三步：在新实例上执行 rsync（本地带 -A 转发登录）

```bash
# -A 参数：开启 SSH Agent 转发，让新实例能用你本地的密钥去连旧实例
ssh -A autodl-new

# ── 进入新实例后执行 ──

# 替换下面的 <旧实例内网IP> 和 <旧实例SSH端口>
# 旧实例端口从 ~/.ssh/config 里的 autodl 配置查
OLD_IP=<旧实例内网IP>
OLD_PORT=<旧实例SSH端口>    # 例如 36153

# 开始 rsync（51GB，内网预计 5-15 分钟）
rsync -avz --progress \
  --exclude='*.pyc' \
  --exclude='__pycache__' \
  --exclude='.cache' \
  -e "ssh -p $OLD_PORT -o StrictHostKeyChecking=no" \
  root@$OLD_IP:/root/autodl-tmp/ \
  /root/autodl-tmp/

# 传输完成后验证
du -sh /root/autodl-tmp/*
```

#### 如果内网不通（备用：本地中转）

```bash
# 本地执行，速度受本地带宽限制（慢，慎用）
rsync -avz --progress \
  -e "ssh -p <旧端口>" \
  root@<旧公网IP>:/root/autodl-tmp/ \
  - | ssh -p <新端口> root@<新公网IP> 'rsync -avz --files-from=- / /'

# 或者更简单的 tar 中转
ssh autodl 'cd /root && tar czf - autodl-tmp/' | ssh autodl-new 'cd /root && tar xzf -'
```

### Step 4：重建软链接（新实例，必做）

```bash
# 登录新实例
ssh autodl-new

# 重建 models 软链接
ln -sf /root/autodl-tmp/models /root/models

# FlashHead 模型路径软链
ln -sf /root/models /root/services/flashhead/models

# Singer 模型路径软链
mkdir -p /root/services/singer/pretrained_models
ln -sf /root/models/SoulX-Singer /root/services/singer/pretrained_models/SoulX-Singer
ln -sf /root/models/SoulX-Singer-Preprocess /root/services/singer/pretrained_models/SoulX-Singer-Preprocess

# 验证
ls -la /root/models
ls -la /root/services/flashhead/models
ls -la /root/services/singer/pretrained_models/
```

---

### Step 5：安装 flash_attn（A100 有预编译 wheel，约 5 分钟）

```bash
ssh autodl-new

source ~/miniconda3/etc/profile.d/conda.sh
conda activate /root/autodl-tmp/envs/env-flashhead

# A100 支持预编译 wheel，无需源码编译！
pip install flash_attn \
  -i https://pypi.tuna.tsinghua.edu.cn/simple \
  --timeout 60 --retries 3

# 验证
python -c "import flash_attn; print('flash_attn:', flash_attn.__version__)"
```

---

### Step 6：启动所有服务并验证

```bash
ssh autodl-new 'bash ~/start_all.sh'

# 等 30 秒后验证
sleep 30
ssh autodl-new '
  echo "=== FlashHead ===" && tail -5 ~/logs/flashhead.log
  echo "=== Podcast ===" && tail -3 ~/logs/podcast.log
  echo "=== Singer ===" && tail -5 ~/logs/singer.log
  echo "=== 端口 ===" && cat /proc/net/tcp | awk "NR>1{printf \"%d\n\", strtonum(\"0x\" substr(\$2,7))}" | grep -E "6006|6008|7862"
  echo "=== GPU ===" && nvidia-smi | grep MiB
'
```

**新实例公网地址：**（需在 AutoDL 自定义服务里查看）
- FlashHead：6006 → 新的 `https://xxxx.westd.seetacloud.com:8443`
- Podcast：6008 → 新的 `https://yyyy.westd.seetacloud.com:8443`

---

### Step 7：确认无误后关闭旧实例

```
AutoDL 控制台 → 旧实例 → 「关机」→ 确认释放
```
