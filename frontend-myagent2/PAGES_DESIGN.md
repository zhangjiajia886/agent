# Agent 编排平台 - 前端管理页面设计

> 对标 Claude Code 全部实体模块，前端覆盖所有管理维度
> 审核通过后开始实施

---

## 一、页面总览与导航结构

```
┌───────────────────────────────────────────────────────────────┐
│ AgentFlow    [工作流]  [技能]  [模型]  [工具]  [MCP]  ...  👤 │
├─────┬─────────────────────────────────────────────────────────┤
│     │                                                         │
│ 侧  │                    主内容区                              │
│ 边  │                                                         │
│ 栏  │                                                         │
│     │                                                         │
└─────┴─────────────────────────────────────────────────────────┘
```

### 路由设计

| 路由 | 页面 | 对标 Claude Code 模块 | 说明 |
|------|------|----------------------|------|
| `/workflows` | 工作流管理 | 核心执行引擎 | 工作流列表、新建、版本管理 |
| `/workflows/:id/edit` | 画布编辑器 | query.ts 主循环 | 当前已实现的拖拽画布 |
| `/workflows/:id/runs` | 执行历史 | 执行日志 | 运行记录、Trace 瀑布图 |
| `/skills` | Skill 管理 | `skills/*.md` | Markdown 技能模板 CRUD |
| `/prompts` | Prompt 模板库 | System Prompt 模板 | 可复用 Prompt 片段管理 |
| `/models` | 模型配置 | LLM Provider 配置 | 多模型多 Provider 管理 |
| `/tools` | 工具管理 | `Tool.ts` 40+ 内置工具 | 内置工具+自定义工具注册 |
| `/mcp` | MCP Server 管理 | MCP 协议集成 | Server 连接、工具发现 |
| `/secrets` | 密钥管理 | 环境变量/凭证 | 加密存储敏感信息 |
| `/permissions` | 权限策略 | 三级权限控制 | 工具执行权限配置 |
| `/knowledge` | 知识库 | CLAUDE.md / 项目记忆 | 文件/文本知识片段管理 |
| `/executions` | 全局执行监控 | 运行时状态 | 跨工作流运行总览 |
| `/settings` | 系统设置 | 全局配置 | 资源限制、预算、沙箱模式 |

### 2.2 Skill 管理 `/skills`（对标 Claude Code `skills/*.md`）

Claude Code 的 Skill 是 Markdown 文件，定义 Agent 的"能力模板"——一段结构化的指令，可被嵌入到任意 LLM 节点的 System Prompt 中。

```
┌──────────────────────────────────────────────────────────────────┐
│ 🎯 Skill 管理（技能模板）                    [+ 新建 Skill]      │
├──────────────────────────────────────────────────────────────────┤
│ 分类: [全部] [编程] [数据分析] [文本处理] [自定义]                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ ┌──────────────────────────────────────────────────────────────┐│
│ │ 📄 SQL 专家                                   [编程] [内置]  ││
│ │ 根据自然语言生成安全的 SELECT 查询                            ││
│ │ 引用次数: 12 · 最后修改: 3天前                                ││
│ │ 依赖工具: sql_query, read_file                               ││
│ │                                        [编辑] [复制] [插入↗] ││
│ ├──────────────────────────────────────────────────────────────┤│
│ │ 📄 代码审查                              [编程] [自定义]      ││
│ │ 审查代码质量，检查安全漏洞和性能问题                          ││
│ │ 引用次数: 8 · 最后修改: 1天前                                 ││
│ │ 依赖工具: read_file, grep_search, bash                       ││
│ │                                        [编辑] [复制] [插入↗] ││
│ ├──────────────────────────────────────────────────────────────┤│
│ │ 📄 数据清洗                             [数据分析] [自定义]   ││
│ │ 处理 CSV/JSON 数据，去重、填充空值、格式化                    ││
│ │                                        [编辑] [复制] [插入↗] ││
│ └──────────────────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────────────────┘
```

**Skill 编辑器**（全屏 Monaco Editor + 预览）：
```
┌──────────────────────────────────────────────────────────────────┐
│ ✏️ 编辑 Skill: SQL 专家             [预览] [保存] [另存为模板]   │
├──────────┬───────────────────────────────────────────────────────┤
│ 元信息    │  Markdown 编辑器 (Monaco)                            │
│          │  ┌───────────────────────────────────────────────────┐│

│ [SQL专家]│  │                                                   ││
│          │  │ ## 角色                                            ││
│ 分类:    │  │ 你是一个 SQL 查询专家，擅长将自然语言转换为...     ││
│ [编程 ▼] │  │                                                   ││
│          │  │ ## 约束                                            ││

│ [sql]    │  │ - 使用 readonly 连接                               ││
│ [数据库] │  │ - 必须处理 NULL 值                                 ││
│          │  │                                                   ││
│ 依赖工具:│  │ ## 输出格式                                       ││
│ ☑sql_qry│  │ 返回纯 SQL，不要 markdown 代码块                  ││
│ ☑rd_file│  │                                                   ││
│ ☐bash   │  │ ## 示例                                            ││
│          │  │ 输入: "上个月销售额最高的产品"                      ││
│ 变量:    │  │ 输出: SELECT p.name, SUM(o.amount)...             ││
│ {{schema}}│ │                                                   ││
│ {{query}} │ └───────────────────────────────────────────────────┘│
└──────────┴───────────────────────────────────────────────────────┘
```

**数据模型**:
```typescript
interface Skill {
  id: string;
  name: string;
  description: string;
  category: 'coding' | 'data_analysis' | 'text_processing' | 'custom';
  tags: string[];
  content: string;              // Markdown 全文
  variables: string[];          // 提取的 {{变量名}} 列表
  requiredTools: string[];      // 依赖的工具
  source: 'builtin' | 'user';  // 内置/用户自定义
  referenceCount: number;       // 被多少个 LLM 节点引用
  createdAt: string;
  updatedAt: string;
}
```

### 2.3 Prompt 模板库 `/prompts`

│  ┌────────────────┐ ┌────────────────┐ ┌────────────────┐       │

│  │  助的助手..."  │ │  格式输出..."  │ │  考问题..."    │       │
│  │ [插入] [编辑]   │ │ [插入] [编辑]   │ │ [插入] [编辑]   │       │

│   ├── /secrets            → SecretsPage
│   ├── /permissions        → PermissionPage
│   ├── /knowledge          → KnowledgePage
│   ├── /executions         → ExecutionMonitorPage

### 4.2 新增依赖

```
react-router-dom    — 路由

### 4.3 实施顺序（建议）

```
Phase 1: 骨架 + 核心管理
