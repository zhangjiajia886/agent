# TTS 应用系统设计文档

## 📋 需求分析

### 核心需求
1. **后端框架**：Python FastAPI（为后期集成 RAG、KG、Agent 预留扩展性）
2. **前端框架**：ele-admin-plus-ts（Vue3 + TypeScript + Element Plus）
3. **核心功能**：基于 Fish Speech API 的 TTS/ASR 语音服务
4. **数据存储**：MySQL + Redis + Elasticsearch + Milvus
5. **UI/UX**：现代化界面，配图完善

### 扩展规划
- **RAG（检索增强生成）**：知识库问答、文档理解
- **KG（知识图谱）**：实体关系管理、语义搜索
- **Agent（智能代理）**：多步骤任务执行、工具调用

## 🗄️ 数据库设计

### MySQL 表结构

#### 1. users（用户表）
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(100),
    avatar_url VARCHAR(255),
    is_active BOOLEAN DEFAULT TRUE,
    is_superuser BOOLEAN DEFAULT FALSE,
    fish_api_key VARCHAR(255),           -- Fish Speech API Key
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 2. voice_models（声音模型表）
```sql
CREATE TABLE voice_models (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    fish_model_id VARCHAR(100) UNIQUE NOT NULL,  -- Fish Speech 模型 ID
    title VARCHAR(100) NOT NULL,
    description TEXT,
    language VARCHAR(10) DEFAULT 'zh',
    visibility ENUM('private', 'public') DEFAULT 'private',
    sample_audio_url VARCHAR(255),               -- 参考音频 URL
    usage_count INT DEFAULT 0,
    is_deleted BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id),
    INDEX idx_fish_model_id (fish_model_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3. tts_tasks（TTS 任务表）
```sql
CREATE TABLE tts_tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    voice_model_id BIGINT,
    text TEXT NOT NULL,                          -- 合成文本
    format ENUM('mp3', 'wav', 'pcm', 'opus') DEFAULT 'mp3',
    latency ENUM('normal', 'balanced', 'low') DEFAULT 'balanced',
    streaming BOOLEAN DEFAULT FALSE,
    audio_url VARCHAR(255),                      -- 生成的音频文件 URL
    audio_size INT,                              -- 文件大小（字节）
    duration FLOAT,                              -- 音频时长（秒）
    status ENUM('pending', 'processing', 'completed', 'failed') DEFAULT 'pending',
    error_message TEXT,
    cost_credits DECIMAL(10, 2),                 -- 消耗积分
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (voice_model_id) REFERENCES voice_models(id) ON DELETE SET NULL,
    INDEX idx_user_id (user_id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 4. asr_tasks（ASR 任务表）
```sql
CREATE TABLE asr_tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    audio_url VARCHAR(255) NOT NULL,             -- 上传的音频文件 URL
    language VARCHAR(10) DEFAULT 'zh',
    recognized_text TEXT,                        -- 识别结果
    duration FLOAT,                              -- 音频时长
    segments JSON,                               -- 分段详情（时间戳）
    status ENUM('pending', 'processing', 'completed', 'failed') DEFAULT 'pending',
    error_message TEXT,
    cost_credits DECIMAL(10, 2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 5. knowledge_bases（知识库表 - RAG 扩展）
```sql
CREATE TABLE knowledge_bases (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    type ENUM('document', 'qa', 'custom') DEFAULT 'document',
    embedding_model VARCHAR(50) DEFAULT 'text-embedding-3-small',
    chunk_size INT DEFAULT 500,
    chunk_overlap INT DEFAULT 50,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 6. documents（文档表 - RAG 扩展）
```sql
CREATE TABLE documents (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    knowledge_base_id BIGINT NOT NULL,
    title VARCHAR(200) NOT NULL,
    content LONGTEXT NOT NULL,
    file_url VARCHAR(255),
    file_type VARCHAR(50),
    chunk_count INT DEFAULT 0,
    vector_ids JSON,                             -- Milvus 向量 ID 列表
    metadata JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (knowledge_base_id) REFERENCES knowledge_bases(id) ON DELETE CASCADE,
    INDEX idx_kb_id (knowledge_base_id),
    FULLTEXT idx_content (content)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 7. agent_tasks（Agent 任务表 - 扩展）
```sql
CREATE TABLE agent_tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    task_type VARCHAR(50) NOT NULL,              -- 任务类型
    input_data JSON NOT NULL,                    -- 输入数据
    output_data JSON,                            -- 输出结果
    steps JSON,                                  -- 执行步骤记录
    status ENUM('pending', 'running', 'completed', 'failed') DEFAULT 'pending',
    error_message TEXT,
    started_at TIMESTAMP NULL,
    completed_at TIMESTAMP NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 🔌 核心 API 设计

### 1. TTS 接口

#### POST /api/v1/tts/synthesize
```json
// 请求
{
  "text": "你好，我是AI助手",
  "voice_model_id": 123,
  "format": "mp3",
  "latency": "balanced",
  "streaming": false
}

// 响应
{
  "task_id": 456,
  "audio_url": "https://cdn.example.com/audio/456.mp3",
  "duration": 3.5,
  "size": 56789,
  "cost_credits": 0.05
}
```

#### WebSocket /api/v1/tts/stream
```json
// 客户端发送
{
  "event": "start",
  "text": "实时流式合成测试",
  "voice_model_id": 123,
  "format": "mp3"
}

// 服务端推送（二进制音频流）
Binary data chunks...

// 完成通知
{
  "event": "finish",
  "duration": 2.3,
  "cost_credits": 0.03
}
```

### 2. ASR 接口

#### POST /api/v1/asr/recognize
```json
// 请求（multipart/form-data）

```

## 🎨 前端页面设计

### 1. TTS 语音合成页面
- **输入区域**：文本编辑器（支持多行、字数统计）
- **声音选择**：下拉选择器 + 试听按钮
- **参数配置**：格式、延迟模式、流式开关
- **操作按钮**：合成、下载、播放
- **历史记录**：表格展示历史任务

### 2. ASR 语音识别页面
- **上传区域**：拖拽上传 + 录音功能
- **识别结果**：文本展示 + 时间轴
- **操作按钮**：识别、复制、导出
- **历史记录**：表格展示历史任务

### 3. 声音模型管理页面
- **模型列表**：卡片式展示 + 搜索过滤
- **创建模型**：上传音频 + 表单填写

## 📦 部署方案

### Docker Compose 部署

```yaml
services:
  # 后端服务
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    depends_on:
      - mysql
      - redis
      - elasticsearch
      - milvus
    environment:
      - DATABASE_URL=mysql+aiomysql://user:pass@mysql:3306/ttsapp
      - REDIS_URL=redis://:redis123456@redis:6379/0
      - FISH_API_KEY=${FISH_API_KEY}

  # 前端服务
  frontend:
    build: ./frontend
    ports:
      - "3000:80"
    depends_on:
      - backend

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - backend
      - frontend
```
