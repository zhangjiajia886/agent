# F6 Trace 时间线设计

> 状态：待实施  
> 对应后端：P6 恢复与观测

---

## 1. 背景

后端 EventTracer 记录了 LLM 调用、工具调用、步骤状态变化等全链路 Trace。`GET /tasks/{uid}/trace` 返回完整时间线。前端需要时间线视图展示这些数据。

## 2. 目标

- [ ] Trace 时间线视图：按时间排列展示所有执行事件。
- [ ] 工具调用详情：展示工具输入/输出、耗时、状态。
- [ ] LLM 调用详情：展示 model、token 消耗。
- [ ] 筛选和搜索：按 trace_type 或 tool_name 过滤。

## 3. 涉及文件

| 文件 | 类型 | 说明 |
|---|---|---|
| `components/TraceTimeline.vue` | 新增 | Trace 时间线主组件 |
| `components/TraceItem.vue` | 新增 | 单条 Trace 项 |
| `views/comic-agent/TaskTrace.vue` | 新增 | Trace 详情页 |

## 4. API

```typescript
const trace = await api.get(`/comic-agent/tasks/${taskUid}/trace`)
```
