# F1 任务工作台：基础知识讲解

## 1. 这个技术是什么

任务工作台是 Agent 前端的核心 UI 组件，负责将后端的任务状态（steps/artifacts/report）可视化展示给用户。类似 IDE 的构建输出面板或 CI/CD 的 pipeline 视图。

## 2. 为什么要用

- 用户需要看到 Agent 在做什么（不是黑盒）。
- 多步骤任务需要进度追踪。
- 产物（图片/视频/音频）需要即时预览。
- 失败步骤需要明确展示原因。

## 3. 好处

- **透明度**：用户随时知道 Agent 执行到哪一步。
- **信任感**：看到步骤逐步完成，比等待黑盒更好。
- **可操作性**：失败步骤可以重试。
- **产物即时预览**：不用等全部完成才能看到中间结果。

## 4. 弊端

- **组件复杂度**：需要处理多种事件和状态变化。
- **实时性要求**：WebSocket 断连时需要优雅降级。
- **移动端适配**：步骤面板在小屏幕上需要折叠。

## 5. 竞争技术 / 替代方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 纯文本 log | 简单 | 不直观 |
| 进度条 | 轻量 | 看不到步骤细节 |
| Pipeline 视图 | 直观 | 实现复杂 |
| 卡片式面板 | 美观 | 占空间大 |
| 时间线视图 | 适合追踪 | 步骤多时太长 |

## 7. 关键点

任务工作台不是单独页面，而是嵌入在聊天界面右侧或下方的面板，随 WebSocket 事件实时更新。

```text
Event-driven UI ≠ Polling UI
```
