# F1 任务工作台设计

> 状态：✅ 已完成  
> 对应后端：P0 状态事件 / P3 Scheduler  
> 完成日期：2026-05-01

---

## 1. 背景

当前前端 `ComicAgentView.vue` 是单体文件，所有任务状态展示逻辑混在一起。需要拆分为独立的任务工作台组件。

## 2. 目标

- [ ] 独立 `TaskWorkspace` 组件，接收 WebSocket 事件驱动状态。
- [ ] 步骤面板：展示所有 steps 及其状态（pending/running/succeeded/failed/blocked）。
- [ ] 产物画廊：展示 artifact_created 事件产出的图片/视频/音频。
- [ ] 最终报告：展示 done.final_report 的完成摘要。
- [ ] 步骤进度条：按 steps 完成比例展示进度。

## 3. 涉及文件

| 文件 | 类型 | 说明 |
|---|---|---|
| `ComicAgentView.vue` | 修改 | 提取任务工作台逻辑 |
| `components/TaskWorkspace.vue` | 新增 | 任务工作台主组件 |
| `components/StepPanel.vue` | 新增 | 步骤状态面板 |
| `components/ArtifactGallery.vue` | 新增 | 产物画廊 |

## 4. 事件消费

```text
WebSocket 事件流
  ↓
task_created → 初始化 steps 面板
step_update  → 更新对应 step 状态和描述
tool_start   → 显示工具执行中 spinner
tool_done    → 更新工具结果
artifact_created → 追加到产物画廊
done         → 显示最终报告
```

## 5. 数据结构

```typescript
interface TaskStep {
  step_uid: string
  title: string
  description: string
  tool_name: string | null
  status: 'pending' | 'running' | 'succeeded' | 'failed' | 'blocked' | 'canceled'
  inputs: Record<string, any>
  outputs: Record<string, any>
  error: Record<string, any> | null
}

interface TaskState {
  task_uid: string
  user_goal: string
  status: string
  steps: TaskStep[]
  artifacts: Artifact[]
  current_step_uid: string | null
}
```
