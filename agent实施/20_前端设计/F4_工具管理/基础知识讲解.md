# F4 工具管理：基础知识讲解

## 1. 这个技术是什么

工具管理面板是 Agent 系统的运维视图，展示所有可用工具的元信息、健康状态和历史统计。类似 Kubernetes Dashboard 的 Pod 状态页。

## 2. 为什么要用

- 运维人员需要知道哪些工具可用、哪些异常。
- 用户想了解 Agent 具备哪些能力。
- 工具调用统计有助于发现问题（某工具失败率突增）。

## 3. 好处

- **可见性**：一目了然所有工具状态。
- **问题发现**：失败率异常时及时告警。
- **能力展示**：新用户快速了解 Agent 能力范围。

## 4. 弊端

- **维护成本**：新增工具时需要更新元数据。
- **数据量**：统计查询可能较慢。

## 5. 竞争技术 / 替代方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 不展示 | 简单 | 黑盒 |
| 简单列表 | 轻量 | 信息少 |
| 卡片式面板 | 美观 | 实现工作量大 |
| 监控 Dashboard | 专业 | 过重 |

## 6. 图示

```text
GET /tools/health → 工具列表 + 状态
GET /tools/stats  → 调用次数 + 成功率
  ↓
ToolCard × N
  ↓
每个卡片展示: 名称 / 风险 / 成本 / 状态 / 统计 / fallback
```
