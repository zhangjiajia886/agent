# F2 事件回放设计

> 状态：待实施  
> 对应后端：P6 恢复与观测

---

## 1. 背景

漫剧任务耗时长（几十秒到几分钟），WebSocket 可能断连。用户刷新页面或重连后，需要从历史事件重建工作台状态。

## 2. 目标

- [ ] WebSocket 断连后自动重连。
- [ ] 重连后通过 `GET /tasks/{uid}/events` 拉取历史事件。
- [ ] 逐条 replay 历史事件，重建 TaskWorkspace 状态。
- [ ] 不重新执行工具，只重放状态变化。

## 3. 涉及文件

| 文件 | 类型 | 说明 |
|---|---|---|
| `composables/useEventReplay.ts` | 新增 | 事件回放逻辑 |
| `composables/useWebSocket.ts` | 新增/修改 | 自动重连 + replay |
| `components/TaskWorkspace.vue` | 修改 | 接收 replay 事件 |

## 4. 回放流程

```text
WebSocket 断连
  ↓
自动重连（指数退避）
  ↓
GET /tasks/{uid}/events
  ↓
按时间排序
  ↓
逐条 dispatch 到 TaskWorkspace
  ↓
UI 恢复到最新状态
```

## 5. 核心逻辑

```typescript
async function replayEvents(taskUid: string): Promise<void> {
  const events = await api.getTaskEvents(taskUid)
  for (const event of events) {
    // 不触发动画，静默更新
    taskStore.dispatch(event, { silent: true })
  }
}
```
