# F3 审批交互设计

> 状态：待实施  
> 对应后端：P5 Replanner / P7 沙箱与安全

---

## 1. 背景

后端通过 Sandbox 和 Replanner 判断工具是否需要用户审批。当 `tool_confirm` 事件到达前端时，需要展示审批对话框，让用户 approve 或 reject。

Replanner 的 `ask_user` 决策也需要前端展示问题和选项。

## 2. 目标

- [ ] 工具审批弹窗：显示工具名称、参数、风险等级，支持 approve/reject。
- [ ] ask_user 对话：显示问题和选项列表，用户选择后回传。
- [ ] 审批超时处理：5 分钟无响应自动 reject。
- [ ] 审批历史：显示本次任务的审批记录。

## 3. 涉及文件

| 文件 | 类型 | 说明 |
|---|---|---|
| `components/ToolApproval.vue` | 新增 | 工具审批弹窗 |
| `components/AskUserDialog.vue` | 新增 | Replanner ask_user 对话框 |
| `composables/useApproval.ts` | 新增 | 审批状态管理 |

## 4. 事件流

```text
后端 → tool_confirm 事件
  ↓
前端弹出审批对话框
  ↓
用户点击 approve / reject
  ↓
WebSocket 发送 { action: "approve" } 或 { action: "reject", reason: "..." }
  ↓
后端继续执行或跳过
```

## 5. 审批弹窗数据

```typescript
interface ToolConfirmEvent {
  type: 'tool_confirm'
  task_uid: string
  step_uid: string
  tool: string
  input: Record<string, any>
  description: string
  tool_call_id: string
}
```
