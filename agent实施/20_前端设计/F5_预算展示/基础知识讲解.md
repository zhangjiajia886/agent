# F5 预算展示：基础知识讲解

## 1. 这个技术是什么

预算展示是将 Agent 任务的资源消耗（Token、API 调用、GPU 时间）可视化，帮助用户了解每次任务的成本。类似云服务的费用仪表盘。

## 2. 为什么要用

- 用户需要了解任务成本（尤其是付费场景）。
- 帮助用户优化提示词，减少不必要的工具调用。
- 运营需要统计全局资源消耗。

## 3. 好处

- **透明**：用户知道钱花在哪里。
- **优化**：发现高成本操作，引导用户优化。
- **预警**：接近配额时提前通知。

## 4. 弊端

- **估算不精确**：成本是预估值。
- **可能引起焦虑**：用户看到成本数字可能不敢操作。

## 5. 竞争技术 / 替代方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 不展示成本 | 简单 | 不透明 |
| 仅展示 Token | 轻量 | 不完整 |
| 完整成本面板 | 透明 | 实现复杂 |
| 后台统计不面向用户 | 无焦虑 | 不透明 |

## 6. 图示

```text
done.metadata.budget_usage
  ↓
BudgetSummary 组件
  ↓
Token 进度条 + 工具调用列表 + 预估成本
```
