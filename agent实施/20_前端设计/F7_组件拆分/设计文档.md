# F7 组件拆分设计

> 状态：待实施  
> 对应后端：—（纯前端架构）

## 1. 背景

当前 `ComicAgentView.vue` 是一个 1000+ 行的单体文件，所有逻辑（聊天、任务面板、审批、产物展示）混在一起。需要拆分为可维护的子组件和 composable。

## 2. 目标

- [ ] 将 ComicAgentView 拆分为子组件。
- [ ] 提取 composable（useChat、useTask、useWebSocket）。
- [ ] 建立清晰的组件通信机制。
- [ ] 保持现有功能不受影响。

## 3. 组件树

```text
ComicAgentView.vue (容器)
├── ChatPanel.vue (聊天面板)
│   ├── MessageList.vue
│   ├── MessageInput.vue
│   └── ThinkingIndicator.vue
├── TaskWorkspace.vue (任务工作台)
│   ├── StepPanel.vue
│   ├── ArtifactGallery.vue
│   └── FinalReport.vue
│       └── BudgetSummary.vue
├── ToolApproval.vue (审批弹窗)
├── AskUserDialog.vue (ask_user 对话)
└── Sidebar.vue (侧边栏)
    ├── ConversationList.vue
    └── ModelSelector.vue
```

## 4. Composable 拆分

```text
composables/
├── useChat.ts          # 消息管理、发送、历史
├── useTask.ts          # 任务状态管理、步骤追踪
├── useWebSocket.ts     # WebSocket 连接、事件分发
├── useEventReplay.ts   # 断线恢复、事件回放
├── useApproval.ts      # 审批状态管理
└── useArtifacts.ts     # 产物收集和预览
```

## 5. 拆分原则

- **单一职责**：每个组件只做一件事。
- **Props Down, Events Up**：父子通信用 props + emit。
- **Composable 共享状态**：跨组件状态放 composable。
- **渐进式拆分**：一次拆一个，每次拆完验证。
