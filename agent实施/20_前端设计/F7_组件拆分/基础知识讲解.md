# F7 组件拆分：基础知识讲解

## 1. 这个技术是什么

组件拆分是将大型 Vue 单文件组件（SFC）分解为多个小型、单一职责的子组件和 composable 的架构重构过程。

## 2. 为什么要用

- 单体组件超过 500 行后难以维护。
- 逻辑耦合导致修改一处影响多处。
- 多人协作时容易冲突。
- 难以复用和测试。

## 3. 好处

- **可维护性**：每个组件 100-300 行，易读易改。
- **可测试性**：小组件可以单独测试。
- **可复用性**：通用组件可跨页面使用。
- **开发效率**：多人可并行开发不同组件。

## 4. 弊端

- **过度拆分**：太细碎会增加组件通信成本。
- **Props 钻透**：层级太深导致 props 传递链过长。
- **迁移风险**：拆分过程中可能引入回归 bug。

## 5. 竞争技术 / 替代方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 保持单文件 | 无迁移成本 | 越来越难维护 |
| 只提取 composable | 改动小 | template 仍然臃肿 |
| 完整组件拆分 | 架构清晰 | 工作量大 |
| Pinia Store | 集中状态管理 | 引入新依赖 |

## 6. 图示

```text
Before:
  ComicAgentView.vue (1500 lines)

After:
  ComicAgentView.vue (200 lines, 容器)
    ├── ChatPanel.vue (300 lines)
    ├── TaskWorkspace.vue (250 lines)
    ├── ToolApproval.vue (150 lines)
    └── composables/ (状态逻辑)
```

## 7. 关键点

拆分不是一次性完成的，应该渐进式地从最独立的模块开始。

```text
Refactor incrementally, not all at once
```
