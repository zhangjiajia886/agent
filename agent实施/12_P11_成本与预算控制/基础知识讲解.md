# P11 成本与预算控制：基础知识讲解

## 1. 这个技术是什么

成本与预算控制是限制 Agent 在一次任务、一个用户、一个时间窗口内能消耗多少资源的机制。

资源包括：

- LLM token
- ComfyUI GPU 时间
- 即梦 API 调用
- 视频生成次数
- TTS 次数
- 存储空间
- 任务总耗时

## 2. 为什么要用预算控制

漫剧 Agent 的工具不是普通函数，很多调用都有真实成本。

没有预算控制时：

```text
LLM 重复失败重试
  ↓
反复生成图片/视频
  ↓
GPU/API 成本失控
```

## 3. 好处

- **防止无限消耗**：限制重试和工具调用次数。
- **保护用户额度**：避免单任务消耗过高。
- **提高稳定性**：预算耗尽后转入 ask_user/block。
- **便于统计**：知道哪个工具最贵、哪个模型最耗。

## 4. 弊端

- **可能提前阻止任务**：预算过小会影响体验。
- **成本估算不一定准确**。
- **需要维护不同工具的成本模型**。
- **需要和套餐/用户等级联动**。

## 5. 竞争技术 / 替代方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 不限制 | 简单 | 成本失控 |
| 固定工具次数限制 | 易实现 | 不区分工具成本 |
| Token 预算 | 控 LLM 成本 | 不控 GPU/API |
| 综合 BudgetController | 全面 | 实现复杂 |
| 账户余额系统 | 商业化强 | 需要完整计费体系 |

## 6. 图示

```text
TaskBudget
  ├── max_tool_calls
  ├── max_image_generations
  ├── max_video_generations
  ├── max_tokens
  ├── max_duration_seconds
  └── max_estimated_cost
          ↓
  每次工具/LLM 调用前检查
          ↓
  allow / ask_user / block
```

## 7. 关键点

预算控制不是为了省钱而阻止任务，而是为了防止失控。

```text
预算耗尽时，应给用户选择：继续消耗 / 降低参数 / 停止任务
```
