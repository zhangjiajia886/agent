# P1 持久化测试用例

## 用例 1：任务落库

- **操作**：执行一次图片生成任务
- **检查**：
  - [ ] `agent_task` 有记录
  - [ ] `status` 最终正确
  - [ ] `final_report` 已写入

## 用例 2：步骤落库

- **检查**：
  - [ ] `agent_step` 有对应 task_uid 的步骤
  - [ ] 工具开始时 running
  - [ ] 工具完成后 succeeded/failed

## 用例 3：产物落库

- **检查**：
  - [ ] `agent_artifact` 有 image/video/audio/file
  - [ ] `url` 可访问
  - [ ] `verified=true`

## 用例 4：事件落库

- **检查**：
  - [ ] `agent_event` 有 task_created
  - [ ] 有 step_update
  - [ ] 有 tool_start/tool_done
  - [ ] 有 done

## 用例 5：工具调用落库

- **检查**：
  - [ ] `tool_invocation` 记录 tool_name/input/output/status
  - [ ] 失败时 error 不为空

## 用例 6：API 查询

- **接口**：
  - [ ] `GET /api/v1/comic-agent/tasks/{task_uid}`
  - [ ] `GET /api/v1/comic-agent/tasks/{task_uid}/events`
  - [ ] `GET /api/v1/comic-agent/tasks/{task_uid}/artifacts`
  - [ ] `GET /api/v1/comic-agent/conversations/{conversation_id}/tasks`
- **权限检查**：
  - [ ] 非任务所属用户访问返回 404
  - [ ] 非会话所属用户访问会话任务列表返回 404

## 用例 7：短事务边界

- **操作**：执行一个包含工具调用的 Agent 任务，并在工具执行过程中断开 WebSocket
- **检查**：
  - [ ] 已开始的 `tool_invocation` 仍然可查询
  - [ ] WebSocket 断开不导致数据库连接长时间占用
  - [ ] 已写入的 `agent_event` 可用于后续回放
