# P0.5 状态闭环验证用例

## 用例 1：普通聊天兼容

- **输入**：你好，你是谁？
- **期望**：
  - [ ] 不调用工具
  - [ ] 不出现错误事件
  - [ ] 前端可显示 assistant 文本

## 用例 2：图片生成任务

- **输入**：生成一张仙侠风格人物图
- **期望事件**：
  - [ ] `task_created`
  - [ ] `step_update: running`
  - [ ] `tool_start`
  - [ ] `tool_done.standard_result`
  - [ ] `artifact_created`
  - [ ] `step_update: succeeded`
  - [ ] `done.final_report`

## 用例 3：模型只列计划不调用工具

- **输入**：生成图片并转视频，请完整执行
- **模拟方式**：使用容易只输出计划的模型或提示
- **期望**：
  - [ ] 前端不误判 completed
  - [ ] 出现 `incomplete` 或 final_report.status 非 completed

## 用例 4：工具失败

- **模拟方式**：关闭 ComfyUI 或提供错误输入
- **期望**：
  - [ ] step 为 failed
  - [ ] final_report.status 为 failed/incomplete/blocked
  - [ ] 前端展示失败原因
