# 实施记录：基础知识讲解

## 1. 这个技术是什么

实施记录是项目变更的工程日志。

它记录：

- 做了什么。
- 修改了哪些文件。
- 执行了哪些命令。
- 验证结果如何。
- 遇到什么问题。
- 如何解决。

## 2. 为什么要用实施记录

Agent 改造周期长，涉及多个阶段。如果不记录，会出现：

```text
不知道改了什么
不知道为什么这样改
不知道怎么验证
不知道失败问题是否已解决
```

实施记录让项目可追踪。

## 3. 好处

- **可复盘**：知道每次变更的原因和结果。
- **可恢复**：部署或重启时知道执行过哪些命令。
- **可交接**：别人能接着做。
- **可审计**：重要改动有记录。

## 4. 弊端

- **需要持续维护**。
- **可能变成流水账**。
- **如果不及时写，会遗漏细节**。

## 5. 竞争技术 / 替代方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 不记录 | 省事 | 后期不可维护 |
| Git commit | 记录代码变化 | 不记录验证过程 |
| Markdown 实施日志 | 灵活清晰 | 需要人工维护 |
| Issue/PR 系统 | 流程规范 | 当前本地项目成本较高 |
| 自动 Trace | 准确 | 需要系统支持 |

## 6. 图示

```text
设计文档 → 代码修改 → 验证命令 → 结果记录 → 问题复盘
                         ↓
                    99_实施记录
```

## 7. 关键点

记录必须包含命令和结果。

例如：

```text
命令：conda run -n ttsapp python -m py_compile ...
结果：通过
备注：后端语法检查通过
```

这样后续重启、部署、排错才有依据。
