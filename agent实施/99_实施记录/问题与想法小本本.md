## 2026-05-01

### 本轮目标

先实现一个可落地版本：

```text
P1 最小持久化底座
  ↓
AgentTaskStore
  ↓
关键事件落库
  ↓
工具调用/产物最小落库
  ↓
保持现有 WebSocket 协议兼容
```

### 问题记录

- `AgentTask.conversation_id` 是非空字段，但 `agent_stream()` 原本没有 `conversation_id/user_id` 参数；P1 最小持久化需要从 WebSocket 层透传 `conv_id/user_id`，否则 `AgentTask` 落库会失败。
- 当前已有 `backend/app/core/comic_chat_agent/task_runtime.py` 单文件，因此不能直接新增同名目录 `task_runtime/store.py`；本轮先使用 `task_store.py` 实现最小 Store，后续 P2/P3 再整体迁移为目录模块。

### 更好的想法

- P1 阶段先只做标准 ReAct Agent 模式的持久化；并行漫剧模式 `ComicOrchestrator` 后续单独接入，避免本轮范围膨胀。
- Store 文件命名从设计稿的 `task_runtime/store.py` 临时调整为 `task_store.py`，这是为了避免破坏现有导入路径。
