# P8 工具治理设计

> 所属阶段：P8  
> 状态：工具能力元数据已落地  
> 目标：建立工具能力元数据、风险等级、审批策略、fallback、健康检查和成本统计体系。

---

## 1. 背景

当前 `ToolRegistry` 已有 name、description、input_schema、executor_type、executor_config 等字段，但 Planner/Scheduler/Replanner 需要更丰富的工具能力信息。

## 2. 目标

- [x] 定义工具能力元数据标准（`ToolCapability` dataclass）。
- [x] 补充风险等级（复用 P7 `RiskLevel`）。
- [x] 补充输入/输出产物类型（`input_artifact_types` / `output_artifact_types`）。
- [x] 补充 `fallback_tools`。
- [x] 补充并行能力（`supports_parallel` / `max_parallel`）。
- [x] 补充成本等级和超时配置（`cost_level` / `timeout_seconds`）。
- [x] 提供查询辅助函数（`get_capability` / `get_fallbacks` / `get_tools_by_output_type` / `get_tools_by_tag`）。
- [ ] 增加工具健康检查（待接入 P9 Redis 健康缓存）。
- [ ] 增加工具统计指标。

## 3. 非目标

- 不重写所有工具。
- 不引入外部工具市场。
- 不在首期接入 MCP。

## 5. 工具能力元数据

建议首期放入 `ToolRegistry.executor_config.capability`：

```json
{
  "capability": {
    "risk_level": "L1",
    "requires_approval": false,
    "auto_mode_allowed": true,
    "input_artifact_types": ["image"],
    "output_artifact_types": ["video"],
    "fallback_tools": ["jimeng_generate_video"],
    "supports_batch": false,
    "supports_frame": true,
    "supports_parallel": false,
    "max_parallel": 1,
    "required_inputs": ["source_image"],
    "optional_inputs": ["motion_prompt"],
    "cost_level": "high",
    "timeout_seconds": 300
  }
}
```

## 6. 风险等级

| 等级 | 说明 | 示例 |
|---|---|---|
| L0 | 只读安全 | list_dir/read_file 限定目录 |
| L1 | 创作工具 | generate_image/text_to_speech |
| L2 | 写入/外部请求 | write_file/http_request |
| L3 | 命令/代码执行 | bash/python_exec |
| L4 | 禁止工具 | 危险命令、敏感读取 |

## 7. 健康检查

工具健康状态：

```text
healthy
unhealthy
degraded
unknown
```

检查对象：

- ComfyUI `/system_stats`。
- 即梦 API 鉴权配置。
- TTS 服务可用性。
- 本地文件目录可写性。

## 8. 工具统计

建议统计：

- 成功次数。
- 失败次数。
- 平均耗时。
- 最近错误码。
- 成本估算。
- fallback 次数。
